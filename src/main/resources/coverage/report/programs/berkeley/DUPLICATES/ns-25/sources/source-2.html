26 /**
27  * A utility to convert an existing, non replicated JE environment for
28  * replication. This is useful when the user wants to initially prototype and
29  * develop a standalone transactional application, and then add replication as
30  * a second stage.
31  * <p>
32  * JE HA environment log files contain types of log records and metadata used
33  * only by replication. Non replicated environments are lacking that
34  * information and must undergo a one time conversion process to add that
35  * metadata and enable replication. The conversion process is one way. Once an
36  * environment directory is converted, the rules that govern {@link
37  * ReplicatedEnvironment} apply; namely, the directory cannot be opened by a
38  * read/write standalone {@link com.sleepycat.je.Environment}. Only a minimum
39  * amount of replication metadata is added, and the conversion process is not
40  * dependent on the size of the existing directory.
41  * <p>
42  * The conversion process takes these steps:
43  * <ol>
44  * <li> Use {@code DbEnableReplication} to convert an existing environment
45  * directory. {@code DbEnableReplication} can be used as a command line
46  * utility, and must be executed locally on the host which houses the
47  * environment directory. Alternatively, {@code DbEnableReplication} may be
48  * used programmatically through the provided APIs.
49  * </li>
50  * <li> Once converted, the environment directory may be treated as an existing
51  * master node, and can be opened with a {@code ReplicatedEnvironment}. No
52  * helper host configuration is needed.
53  * <li> Additional nodes may be created and can join the group as newly created
54  * replicas, as described in {@code ReplicatedEnvironment}.  Since these new
55  * nodes are empty, they should be configured to use the converted master as
56  * their helper node, and will go through the {@link <a
57  * href="{@docRoot}/../ReplicationGuide/lifecycle.html#lifecycle-nodestartup">
58  * replication node lifecycle</a>} to populate their environment
59  * directories. In this case, there will be data in the converted master that
60  * can only be transferred to the replica through a file copy executed with the
61  * help of a {@link com.sleepycat.je.rep.NetworkRestore}
62  * </li>
63  * </ol>
64  * <p>
65  * For example:
66  * <pre class="code">
67  * // Create the first node using an existing environment 
68  * DbEnableReplication converter = 
69  *     new DbEnableReplication(envDirMars,          // env home dir
70  *                             "UniversalRepGroup", // group name
71  *                             "nodeMars",          // node name
72  *                             "mars:5001");        // node host,port
73  * converter.convert();
74  *
75  * ReplicatedEnvironment nodeMars = new ReplicatedEnvironment(envDirMars, ...);
76  * 
77  * // Bring up additional nodes, which will be initialized from 
78  * // nodeMars.
79  * ReplicationConfig repConfig = null;
80  * try {
81  *     repConfig = new ReplicationConfig("UniversalRepGroup", // groupName
82  *                                       "nodeVenus",         // nodeName
83  *                                       "venus:5008");       // nodeHostPort
84  *     repConfig.setHelperHosts("mars:5001");
85  * 
86  *     nodeVenus = new ReplicatedEnvironment(envDirB, repConfig, envConfig);
87  * } catch (InsufficientLogException insufficientLogEx) {
88  * 
89  *     // log files will be copied from another node in the group
90  *     NetworkRestore restore = new NetworkRestore();
91  *     restore.execute(insufficientLogEx, new NetworkRestoreConfig());
92  *     
93  *     // try opening the node now
94  *     nodeVenus = new ReplicatedEnvironment(envDirVenus, 
95  *                                           repConfig,
96  *                                           envConfig);
97  * }
98  * ...
99  * </pre>
100  */
