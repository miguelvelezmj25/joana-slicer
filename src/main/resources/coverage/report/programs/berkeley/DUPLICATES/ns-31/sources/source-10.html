379      * Sets the bulk-load-secondaries configuration property.  By default this
380      * property is false.
381      *
382      * <p>This property is true to cause the initial creation of secondary
383      * indices to be performed as a bulk load.  If this property is true and
384      * {@link EntityStore#getSecondaryIndex EntityStore.getSecondaryIndex} has
385      * never been called for a secondary index, that secondary index will not
386      * be created or written as records are written to the primary index.  In
387      * addition, if that secondary index defines a foreign key constraint, the
388      * constraint will not be enforced.</p>
389      *
390      * <p>The secondary index will be populated later when the {@code
391      * getSecondaryIndex} method is called for the first time for that index,
392      * or when the store is closed and re-opened with this property set to
393      * false and the primary index is obtained.  In either case, the secondary
394      * index is populated by reading through the entire primary index and
395      * adding records to the secondary index as needed.  While populating the
396      * secondary, foreign key constraints will be enforced and an exception is
397      * thrown if a constraint is violated.</p>
398      *
399      * <p>When loading a primary index along with secondary indexes from a
400      * large input data set, configuring a bulk load of the secondary indexes
401      * is sometimes more performant than updating the secondary indexes each
402      * time the primary index is updated.  The absence of foreign key
403      * constraints during the load also provides more flexibility.</p>
404      *
405      * @param secondaryBulkLoad whether bulk-load-secondaries is used.
406      *
407      * @return 'this'.
408      */
