23 /**
24  * This exception may be thrown by a Replica during the {@link <a
25  * href="{@docRoot}/../ReplicationGuide/lifecycle.html#lifecycle-nodestartup">
26  * replication stream sync-up</a>} phase of startup. It indicates that a syncup
27  * cannot proceed without undoing a number of committed transactions that
28  * exceeds the limit defined by {@link ReplicationConfig#TXN_ROLLBACK_LIMIT}.
29  * <p>
30  * It is rare for committed transactions to be rolled back during a
31  * sync-up. One way this can happen is if a replication group has been
32  * executing with a {@link com.sleepycat.je.Durability} policy that specifies a
33  * {@link com.sleepycat.je.Durability.ReplicaAckPolicy ReplicaAckPolicy} of
34  * NONE.
35  * <p>
36  * When ReplicaAckPolicy.NONE is specified, transactions can commit on the
37  * master without receiving any acknowledgments from replica nodes. Using that
38  * policy, it is possible that if the master node crashes at a given time, and
39  * the group fails over and continues on with a new master, the old master's
40  * environment will have transactions on disk that were never replicated and
41  * received by other nodes. When this old master comes back up and rejoins the
42  * group as a replica, it will have committed transactions that need to be
43  * rolled back.
44  * <p>
45  * If the number of committed transactions to be rolled back is less than or
46  * equal to the limit specified by {@link
47  * ReplicationConfig#TXN_ROLLBACK_LIMIT}, JE will automatically truncate the
48  * environment log to remove the unreplicated transactions, and will throw a
49  * {@link RollbackException}. The application only needs to reinstantiate the
50  * ReplicatedEnvironment and proceed on. If the limit specified by {@link
51  * ReplicationConfig#TXN_ROLLBACK_LIMIT} is exceeded, the application will
52  * receive a RollbackProhibitedException to indicate that manual intervention 
53  * is required.
54  * <p>
55  * The RollbackProhibitedException lets the user interject application specific
56  * processing before the log is truncated. The exception message and getter
57  * methods indicate the number of transactions that must be rolled back, and
58  * the time and id of the earliest targeted transaction, and the user can use
59  * this information to make any desired application adjustments. The
60  * application may then manually truncate the log using {@link
61  * com.sleepycat.je.util.DbTruncateLog}.
62  * <p>
63  * Note that any CommitTokens obtained before restarting this
64  * <code>Replica</code> shouldn't be used after
65  * {@link RollbackProhibitedException} is thrown because the token may no 
66  * longer exist on the current <code>Master</code> node.
67  */

93     private static String makeMessage(String nodeName,
94                                       MatchpointSearchResults searchResults,
95                                       VLSN matchpointVLSN,
96                                       int rollbackTxnLimit,
97                                       boolean rollbackDisabled) {
98         long matchpointLSN = searchResults.getMatchpointLSN();
99         long fileNumber = DbLsn.getFileNumber(matchpointLSN);
100         long fileOffset = DbLsn.getFileOffset(matchpointLSN);
101         StringBuilder str = new StringBuilder();
102 
103         str.append("Node ").append(nodeName).
104             append(" must rollback ").append(searchResults.getRollbackMsg()).
105             append(" in order to rejoin the replication group, but ");
106 
107         if (rollbackDisabled) {
108             str.append("rollbacks are disabled because ").
109                 append("je.rep.txnRollbackDisabled has been set to true. ").
110                 append("Either set je.rep.txnRollbackDisabled to false to ").
111                 append("permit automatic rollback,");
112 
113         } else {
114             str.append(" the transaction rollback limit of ").
115                 append(rollbackTxnLimit).append(" prohibits this. ").
116                 append("Either increase je.rep.txnRollbackLimit to a ").
117                 append("value larger than ").append(rollbackTxnLimit).
118                 append(" to permit automatic rollback,");
119         }
120 
121         str.append(" or manually remove the problematic transactions. ").
122             append("To do manual removal, truncate the log to file ").
123             append(FileManager.getFileName(fileNumber)).
124             append(", offset 0x").append(Long.toHexString(fileOffset)).
125             append(", vlsn ").append(matchpointVLSN).
126             append(" using the directions in ").
127             append("com.sleepycat.je.util.DbTruncateLog.");
128 
129         return str.toString();
130     }
