18 /**
19  * Thrown when a deadlock is detected. When this exception is thrown, JE
20  * detected a deadlock and chose one transaction (or non-transactional
21  * operation), the "victim", to invalidate in order to break the deadlock.
22  * Note that this is different than a {@link LockTimeoutException lock
23  * timeout} or {@link TransactionTimeoutException}, which occur for other
24  * reasons.
25  *
26  * <p>For more information on deadlock detection, see
27  * {@link EnvironmentConfig#LOCK_DEADLOCK_DETECT}. As described there, a
28  * {@code DeadlockException} is normally thrown when a random victim is
29  * selected; in this case the exception message will contain the string:
30  * {@code This locker was chosen randomly as the victim}. If the deadlock
31  * exception is thrown in a non-victim thread, due to live lock or an
32  * unresponsive thread, the message will contain the string:
33  * {@code Unable to break deadlock using random victim selection within the
34  * timeout interval}.</p>
35  *
36  * <p>TODO: describe how to debug using info included with the exception.</p>
37  *
38  * <p>Normally, applications should catch the base class {@link
39  * LockConflictException} rather than catching one of its subclasses.  All lock
40  * conflicts are typically handled in the same way, which is normally to abort
41  * and retry the transaction.  See {@link LockConflictException} for more
42  * information.</p>

45  * exception.</p>
46  */
