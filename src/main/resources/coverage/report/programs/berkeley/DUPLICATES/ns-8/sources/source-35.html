16 /**
17  * The result of an operation that successfully reads or writes a record.
18  * <p>
19  * An OperationResult does not contain any failure information. Methods that
20  * perform unsuccessful reads or writes return null or throw an exception. Null
21  * is returned if the operation failed for commonly expected reasons, such as a
22  * read that fails because the key does not exist, or an insertion that fails
23  * because the key does exist.
24  * <p>
25  * Methods that return OperationResult can be compared to methods that return
26  * {@link OperationStatus} as follows: If {@link OperationStatus#SUCCESS} is
27  * returned by the latter methods, this is equivalent to returning a non-null
28  * OperationResult by the former methods.
29  *
30  * @since 7.0
31  */
