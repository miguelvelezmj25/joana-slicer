24 /**
25  * Protocol used by LDiff to request the blocks associated with a database and
26  * do the record level analysis.
27  *
28  * BLOCK LEVEL ANALYSIS
29  * =========================================
30  * The request response sequence for a block list request is:
31  *
32  * EnvDiff -> EnvInfo -> DbBlocks -> BlockListStart [BlockInfo]+ BlockListEnd
33  *
34  * A DbMismatch response is sent back if the database does not exist, or if the
35  * database has different persistent attributes associated with it.
36  *
37  * Note that this request is designed to maximize overlap. That is, the
38  * responder could be working on block n, while the requester is working on
39  * blocks n+1 and beyond (to the end of the database).
40  *
41  * The above is the minimal set of messages, biased for performance when the
42  * databases are expected to match.
43  *
44  * RECORD LEVEL ANALYSIS
45  * =========================================
46  * User can configure LDiff to do record level analysis by setting 
47  * LDiffConfig.setDiffAnalysis(true), it can help you find out which records 
48  * are different between two databases.
49  *
50  * The request response sequence for record level analysis is: 
51  *
52  * RemoteDiffRequest -> DiffAreaStart|Error [RemoteRecord] DiffAreaEnd -> Done 
53  *
54  * The local Environment would send out a RemoteDiffRequest to the remote 
55  * Environment, the remote Environment can get all the records of a different 
56  * area: 
57  *    1. If there exists exceptions during the fetching process, remote 
58  *       Environment would send back an Error message, local Environment would 
59  *       throw out a LDiffRecordRequestException and exit.
60  *    2. If the fetching process is correct, remote Environment would send a 
61  *       DiffAreaStart message to the local Environment to notify that it now
62  *       transfers RemoteRecords of a different area. 
63  *    3. After all the records of a different area are transferred, remote 
64  *       Environment would send out a DiffAreaEnd message, which specifies 
65  *       transferring a different area is finished. 
66  *    4. When all the RemoteDiffRequest are processed, local Environment would
67  *       send a Done message to presents the record level analysis is done.
68  *
69  * TODO: 1) Protocol version matching 2) Block granularity sync request. 3)
70  * Protocol to narrow a diff down to a specific set of key inserts, updates and
71  * deletes.
72  */
