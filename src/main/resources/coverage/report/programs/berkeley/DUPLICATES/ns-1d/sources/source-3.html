22 /**
23  * The methods in this class are disabled if the GlobalCBVLSN is defunct -- see
24  * {@link GlobalCBVLSN}.
25  *
26  * <p>When GlobalCBVLSN} is not defunct, the LocalCBVLSNTracker tracks this
27  * node's local CBVLSN. Each node has a single tracker instance.</p>
28  *
29  * <p>The GlobalCBVLSN must be durable. Since the GlobalCBVLSN is derived from
30  * the LocalCBVLSN, we need to make the LocalCBVLSN durable too. [#18728]</p>
31  * <ol>
32  *     <li>For HA, the GlobalCbvlsn is supposed to ensure that the replication
33  *     stream is always available for replay, across failovers.</li>
34  *</ol>
35  *
36  * <p>The local CBVLSN is maintained by each node. Replicas periodically update
37  * the Master with their current CBVLSN via a response to a heartbeat message
38  * from the Master, where it is managed by the LocalCBVLSNUpdater and
39  * flushed out to RepGroup database, whenever the updater notices that it
40  * has changed. The change is then effectively broadcast to all the Replicas
41  * including the originating Replica, via the replication stream. For this
42  * reason, the CBVLSN for the node as represented in the RepGroup database
43  * will generally lag the value contained in the tracker.</p>
44  *
45  * <p>Note that track() api is invoked in critical code with locks being
46  * held and must be lightweight.</p>
47  *
48  * <p>Local CBVLSNs are used only to contribute to the calculation of the
49  * global CBVLSN. The global CBVLSN acts as the cleaner throttle on old nodes.
50  * Any invariants, such as the rule that the cleaner throttle cannot regress,
51  * are applied when doing the global calculation. In addition, we enforce
52  * the rule against regressing local CBVLSNs here.</p>
53  */
54 public class LocalCBVLSNTracker {
55 
56     /*
57      * Note that all reference fields below are null when then GlobalCBVLSN is
58      * defunct.
59      */
60 
61     /* Used to keep track of the last fsynced matchable VLSN. */
62     private VLSN lastSyncableVLSN;
63 
64     /**
65      * Final syncable VLSN from the penultimate log file.
66      */
67     private VLSN currentLocalCBVLSN;
68 
69     /*
70      * We really only need to update the localCBVLSN once per file. currentFile
71      * is used to determine if this is the first VLSN in the file.
72      */
73     private long currentFile;

76     private boolean allowUpdate = true;

79     private final boolean defunct;

81     public LocalCBVLSNTracker(RepNode repNode, GlobalCBVLSN globalCBVLSN) {
82         defunct = globalCBVLSN.isDefunct();
83 
84         if (!defunct) {
85             VLSNIndex vlsnIndex = repNode.getRepImpl().getVLSNIndex();
86 
87             /* Initialize the currentLocalCBVLSN and lastSyncableVLSN. */
88             currentLocalCBVLSN = vlsnIndex.getRange().getLastSync();
89             lastSyncableVLSN = currentLocalCBVLSN;
90 
91             /* Approximation good enough to start with. */
92             currentFile = DbLsn.getFileNumber(DbLsn.NULL_LSN);
93         }
94     }
95 
96     /* Test hook, disable the LocalCBVLSN updates. */
97     public void setAllowUpdate(boolean allowUpdate) {
98         this.allowUpdate = allowUpdate;
99     }
100 
101     /**
102      * If GlobalCBVLSN is defunct, does nothing.
103      *
104      * <p>If GlobalCBVLSN is defunct, tracks barrier VLSNs, updating the
105      * local CBVLSN if the associated log file has changed. When tracking is
106      * done on a replica, the currentLocalCBVLSN value is ultimately sent
107      * via heartbeat response to the master, which updates the RepGroupDb.
108      * When tracking is done on a master, the update is done on this node.</p>
109      *
110      * <p>The update is only done once per file in order to decrease the cost
111      * of tracking. Since we want the local cbvlsn to be durable, we use the
112      * last vlsn in the penultimate log file as the local cbvlsn value. We know
113      * the penultimate log file has been fsynced, and therefore the last vlsn
114      * within that file has also been fsynced.</p>
115      *
116      * <p>Tracking can be called quite often, and should be lightweight.</p>
117      *
118      * @param newVLSN
119      * @param lsn
120      */
121     public void track(VLSN newVLSN, long lsn) {
122         if (defunct || !allowUpdate) {
123             return;
124         }
125 
126         synchronized (this) {
127             if (newVLSN.compareTo(lastSyncableVLSN) > 0) {
128                 VLSN old = lastSyncableVLSN;
129                 lastSyncableVLSN = newVLSN;
130                 if (DbLsn.getFileNumber(lsn) != currentFile) {
131                     currentFile = DbLsn.getFileNumber(lsn);
132                     currentLocalCBVLSN = old;
133                 }
134             }
135         }
136     }
137 
138     /**
139      * If the GlobalVLSN is not defunct, initialize the local CBVLSN with the
140      * syncup matchpoint, so that the heartbeat responses sent before the node
141      * has replayed any log entries are still valid for saving a place in the
142      * replication stream. If the GlobalVLSN is defunct, do nothing.
143      *
144      * @param matchpoint
145      */
146     public void registerMatchpoint(VLSN matchpoint) {
147         if (defunct) {
148             return;
149         }
150         this.currentLocalCBVLSN = matchpoint;
151         this.lastSyncableVLSN = matchpoint;
152     }
153 
154     /**
155      * @return the local CBVLSN for broadcast from replica to master on the
156      * heartbeat response, or a null VLSN if the GlobalVLSN is defunct.
157      */
158     public VLSN getBroadcastCBVLSN() {
159         return defunct ? VLSN.NULL_VLSN : currentLocalCBVLSN;
160     }
161 
162     /**
163      * @return last syncable VLSN seen by this tracker, or a null VLSN if the
164      * GlobalVLSN is defunct. Note that this VLSN has not yet been broadcast --
165      * see {@link #track}.
166      */
167     public VLSN getLastSyncableVLSN() {
168         return defunct ? VLSN.NULL_VLSN : lastSyncableVLSN;
169     }
