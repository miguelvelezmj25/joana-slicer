27 /**
28  * Specifies the attributes used by a replication {@link Monitor}.
29  * <p>
30  * The following properties identify the target group.
31  * <ul>
32  * <li>groupName: the name of the replication group being monitored. </li>
33  * <li>nodeName: the group-wide unique name associated with this
34  * monitor node.</li>
35  * <li>nodeHost: the hostname and port associated with this Monitor. Used
36  * by group members to contact the Monitor.</li>
37  * <li>helperHosts: the list of replication nodes which the Monitor uses to
38  * register itself so it can receive notifications about group status
39  * changes.</li>
40  * </ul>
41  * The following properties configure the daemon ping thread implemented
42  * within the Monitor. This daemon thread lets the Monitor proactively find
43  * status changes that occur when the Monitor is down or has lost network
44  * connectivity.
45  * <ul>
46  * <li>numRetries: number of times the ping thread attempts to contact a
47  * node before deeming is unreachable. </li>
48  * <li>retryInterval: number of milliseconds between ping thread retries.
49  * </li>
50  * <li>timeout: socketConnection timeout, in milliseconds, specified
51  * when the ping thread attempts to establish a connection with a replication
52  * node. </li>
53  * </ul>
54  * @since JE 5.0
55  */
56 public class MonitorConfig implements Cloneable {

284         int colonToken = hostAndPort.indexOf(":");
285

287             hostAndPort.substring(colonToken + 1) :
