169      * If true, JE HA (replication) will flush all committed transactions to
170      * disk at the specified time interval. This is of interest because the
171      * default durability for replicated transactions of {@link
172      * Durability#COMMIT_NO_SYNC}. The default for this behavior is true.
173      * <p>
174      * When using {@link Durability#COMMIT_NO_SYNC}, continued activity will
175      * naturally cause the steady flush of committed transactions, but a pause
176      * in activity may cause the latest commits to stay in memory. In such a
177      * case, it is unlikely but possible that all members of the replication
178      * group have these last transactions in memory and that no members have
179      * persisted them to disk. A catastrophic failure of all nodes in the
180      * replication group would cause a loss of these transactions, in this
181      * unlikely scenario. This background flush task will reduce such a
182      * possibility.
183      * <p>
184      * Note that enabling this feature when using {@link
185      * Durability#COMMIT_NO_SYNC}, does not constitute a guarantee that
186      * updates made by a transaction are persisted. For an explicit guarantee,

188      * Durability#COMMIT_WRITE_NO_SYNC}. These more stringent, persistent
189      * Durability options can be set at the environment or per-transaction
190      * scope. Using one of these Durability settings for a given transaction
191      * will also flush all commits that occurred earlier in time.
