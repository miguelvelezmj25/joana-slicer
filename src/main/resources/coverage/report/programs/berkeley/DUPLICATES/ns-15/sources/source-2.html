19 /**
20  * Thrown when attempting to use a Database handle that was forcibly closed by
21  * replication.  This exception only occurs in a replicated environment and
22  * normally only occurs on a Replica node.  In the case of a DPL schema upgrade
23  * where an entity class or secondary key is renamed, it may also occur on a
24  * Master node, as described below.
25  *
26  * <p>This exception occurs when accessing a database or store and one of the
27  * following methods was recently executed on the master node and then replayed
28  * on a replica node:
29  * {@link com.sleepycat.je.Environment#truncateDatabase truncateDatabase},
30  * {@link com.sleepycat.je.Environment#removeDatabase removeDatabase} and
31  * {@link com.sleepycat.je.Environment#renameDatabase renameDatabase}.</p>
32  *
33  * <p>When using the {@link com.sleepycat.persist DPL}, this occurs only in two
34  * circumstances:</p>
35  * <ol>
36  * <li>This exception is thrown on a Replica node when the {@link
37  * com.sleepycat.persist.EntityStore#truncateClass truncateClass} method has
38  * been called on the Master node.</li>
39  * <li>This exception is thrown on a Replica or Master node when an entity
40  * class or secondary key has been renamed and the application has been
41  * upgraded. See
42  * <a href="../../persist/evolve/package-summary.html#repUpgrade">Upgrading
43  * a Replication Group</a>.</li>
44  * </ol>
45  *
46  * <p>When this exception occurs, the application must close any open cursors
47  * and abort any open transactions that are using the database or store, and
48  * then close the database or store handle.  If the application wishes, it may
49  * then reopen the database (if it still exists) or store.</p>
50  *
51  * <p>Some applications may wish to coordinate the Master and Replica sites to
52  * prevent a Replica from accessing a database that is being truncated, removed
53  * or renamed, and thereby prevent this exception.  Such coordination is not
54  * directly supported by JE.  The DatabasePreemptedException is provided to
55  * allow an application to handle database truncation, removal and renaming
56  * without such coordination between nodes.</p>
57  *
58  * <p>The {@link com.sleepycat.je.Transaction} handle is <em>not</em>
59  * invalidated as a result of this exception.</p>
60  *
61  * @since 4.0
62  */
