21 /**
22   * VersionedLN is used to  provide an in-memory representation of an LN that
23   * makes its VLSN available through btree access.
24   *
25   * On disk, each log entry is composed of a header (je.log.LogEntryHeader) and
26   * a body (je.log.entry.LogEntry). When an LN is materialized in the Btree, it
27   * usually holds only the body, and does not have access to information in the
28   * log entry header, such as the VLSN. Since version based API operations need
29   * access to the VLSN, environments which are configured with
30   * je.rep.preserveRecordVersion=true instantiate VersionedLNs instead of LNs,
31   * in order to cache the VLSN with the LN, and make it cheaply available to
32   * Btree operations. 
33   */
