42 /**
43  * A store for managing persistent entity objects.
44  *
45  * <p>{@code EntityStore} objects are thread-safe.  Multiple threads may safely
46  * call the methods of a shared {@code EntityStore} object.</p>
47  *
48  * <p>See the <a href="package-summary.html#example">package
49  * summary example</a> for an example of using an {@code EntityStore}.</p>
50  *
51  * <p>Before creating an <code>EntityStore</code> you must create an {@link
52  * Environment} object using the Berkeley DB engine API.  The environment may
53  * contain any number of entity stores and their associated databases, as well
54  * as other databases not associated with an entity store.</p>
55  *
56  * <p>An entity store is based on an {@link EntityModel}: a data model which
57  * defines persistent classes (<em>entity classes</em>), primary keys,
58  * secondary keys, and relationships between entities.  A primary index is
59  * created for each entity class.  An associated secondary index is created for
60  * each secondary key.  The {@link Entity}, {@link PrimaryKey} and {@link
61  * SecondaryKey} annotations may be used to define entities and keys.</p>
62  *
63  * <p>To use an <code>EntityStore</code>, first obtain {@link PrimaryIndex} and
64  * {@link SecondaryIndex} objects by calling {@link #getPrimaryIndex
65  * getPrimaryIndex} and {@link #getSecondaryIndex getSecondaryIndex}.  Then use
66  * these indices to store and access entity records by key.</p>
67  *
68  * <p>Although not normally needed, you can also use the entity store along
69  * with the {@link com.sleepycat.je Base API}.  Methods in the {@link
70  * PrimaryIndex} and {@link SecondaryIndex} classes may be used to obtain
71  * databases and bindings.  The databases may be used directly for accessing
72  * entity records.  The bindings should be called explicitly to translate
73  * between {@link com.sleepycat.je.DatabaseEntry} objects and entity model
74  * objects.</p>
75  *
76  * <p>Each primary and secondary index is associated internally with a {@link
77  * Database}.  With any of the above mentioned use cases, methods are provided
78  * that may be used for database performance tuning.  The {@link
79  * #setPrimaryConfig setPrimaryConfig} and {@link #setSecondaryConfig
80  * setSecondaryConfig} methods may be called anytime before a database is
81  * opened via {@link #getPrimaryIndex getPrimaryIndex} or {@link
82  * #getSecondaryIndex getSecondaryIndex}.  The {@link #setSequenceConfig
83  * setSequenceConfig} method may be called anytime before {@link #getSequence
84  * getSequence} is called or {@link #getPrimaryIndex getPrimaryIndex} is called
85  * for a primary index associated with that sequence.</p>
86  *
87  * <!-- begin JE only -->
88  * <h3>Database Names</h3>
89  *
90  * <p>The database names of primary and secondary indices are designed to be
91  * unique within the environment and identifiable for debugging and use with
92  * tools such as {@link com.sleepycat.je.util.DbDump} and {@link
93  * com.sleepycat.je.util.DbLoad}.</p>
94  *
95  * <p>The syntax of a primary index database name is:</p>
96  * <pre>   persist#STORE_NAME#ENTITY_CLASS</pre>
97  * <p>Where STORE_NAME is the name parameter passed to {@link #EntityStore
98  * EntityStore} and ENTITY_CLASS is name of the class passed to {@link
99  * #getPrimaryIndex getPrimaryIndex}.</p>
100  *
101  * <p>The syntax of a secondary index database name is:</p>
102  * <pre>   persist#STORE_NAME#ENTITY_CLASS#KEY_NAME</pre>
103  * <p>Where KEY_NAME is the secondary key name passed to {@link
104  * #getSecondaryIndex getSecondaryIndex}.</p>
105  *
106  * <p>Although you should never have to construct these names manually,
107  * understanding their syntax is useful for several reasons:</p>
108  * <ul>
109  * <li>Exception messages sometimes contain the database name, from which you
110  * can identify the entity class and secondary key.</li>
111  * <li>If you create other databases in the same environment that are not
112  * part of an <code>EntityStore</code>, to avoid naming conflicts the other
113  * database names should not begin with <code>"persist#"</code>.</li>
114  * <li>If you are using {@link com.sleepycat.je.util.DbDump} or {@link
115  * com.sleepycat.je.util.DbLoad} to perform a backup or copy databases between
116  * environments, knowing the database names can be useful.  Normally you will
117  * dump or load all database names starting with
118  * <code>"persist#STORE_NAME#"</code>.</li>
119  * </ul>
120  *
121  * <p>If you are copying all databases in a store as mentioned in the last
122  * point above, there is one further consideration.  There are two internal
123  * databases that must be kept with the other databases in the store in order
124  * for the store to be used.  These contain the data formats and sequences for
125  * the store:</p>
126  * <pre>   persist#STORE_NAME#com.sleepycat.persist.formats</pre>
127  * <pre>   persist#STORE_NAME#com.sleepycat.persist.sequences</pre>
128  * <p>These databases must normally be included with copies of other databases
129  * in the store.  They should not be modified by the application.</p>
130  *
131  * <p>For example, the following code snippet removes all databases for a given
132  * store in a single transaction.</p>
133  * <pre class="code">
134  *  Environment env = ...
135  *  EntityStore store = ...
136  *  Transaction txn = env.beginTransaction(null, null);
137  *  String prefix = "persist#" + store.getStoreName() + "#";
138  *  for (String dbName : env.getDatabaseNames()) {
139  *      if (dbName.startsWith(prefix)) {
140  *          env.removeDatabase(txn, dbName);
141  *      }
142  *  }
143  *  txn.commit();</pre>
144  *
145  * <!-- end JE only -->
146  *
147  * @author Mark Hayes
148  */
