18 /**
19  * The current state of a replication node and the application this node is
20  * running in.
21  * <p>
22  * This includes the following information:
23  * <p>
24  * <ul>
25  * <li>the replication {@link ReplicatedEnvironment.State state} of this 
26  * node</li>
27  * <li>the name of the current master, as known by this node</li>
28  * <li>the time when this node joined the replication group</li>
29  * <li>the latest transaction end (abort or commit) VLSN on this node</li>
30  * <li>the transaction end (abort or commit) VLSN on the master known by this 
31  * node. The difference between transaction end VLSNs on the master versus on
32  * this node gives an indication of how current this node's data is. The gap
33  * in VLSN values indicates the number of replication records that must be 
34  * processed by this node, to be caught up to the master.</li>
35  * <li>the number of feeders running on this node</li>
36  * <li>the system load average for the last minute</li>
37  * <li>
38  * The appState field is a byte array meant to hold information generated by
39  * the JE HA application, as provided by a registered {@link com.sleepycat.je.rep.AppStateMonitor}.
40  * Users are responsible for serializing and deserializing information for this
41  * field.</li>
42  * </ul>
43  * @since 5.0
44  */
