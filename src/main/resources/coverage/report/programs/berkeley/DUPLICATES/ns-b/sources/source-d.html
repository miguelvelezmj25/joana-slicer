48 /**
49  * Provides an enumeration of all key/data pairs in a database, striving to
50  * fetch in disk order.
51  *
52  * Unlike SortedLSNTreeWalker, for which the primary use case is preload, this
53  * class notifies the callback while holding a latch only if that can be done
54  * without blocking (e.g., when the callback can buffer the data without
55  * blocking).  In other words, while holding a latch, the callback will not be
56  * notified if it might block.  This is appropriate for the DOS
57  * (DiskOrderedCursor) use case, since the callback will block if the DOS queue
58  * is full, and the user's consumer thread may not empty the queue as quickly
59  * as it can be filled by the producer thread.  If the callback were allowed to
60  * block while a latch is held, this would block other threads attempting to
61  * access the database, including JE internal threads, which would have a very
62  * detrimental impact.
63  *
64  * Algorithm
65  * =========
66  *
67  * Terminology
68  * -----------
69  * callback: object implementing the RecordProcessor interface
70  * process: invoking the callback with a key-data pair
71  * iteration: top level iteration consisting of phase I and II
72  * phase I: accumulate LSNs
73  * phase II: sort, fetch and process LSNs
74  *
75  * Phase I and II
76  * --------------
77  * To avoid processing resident nodes (invoking the callback with a latch
78  * held), a non-recursive algorithm is used.  Instead of recursively
79  * accumulating LSNs in a depth-first iteration of the tree (like the
80  * SortedLSNTreeWalker algorithm), level 2 INs are traversed in phase I and
81  * LSNs are accumulated for LNs or BINs (more on this below).  When the memory
82  * or LSN batch size limit is exceeded, phase I ends and all tree latches are
83  * released.  During phase II the previously accumulated LSNs are fetched and
84  * the callback is invoked for each key or key-data pair.  Since no latches are
85  * held, it is permissible for the callback to block.
86  *
87  * One iteration of phase I and II processes some subset of the database.
88  * Since INs are traversed in tree order in phase I, this subset is described
89  * by a range of keys.  When performing the next iteration, the IN traversal is
90  * restarted at the highest key that was processed by the previous iteration.
91  * The previous highest key is used to avoid duplication of entries, since some
92  * overlap between iterations may occur.
93  *
94  * LN and BIN modes
95  * ----------------
96  * As mentioned above, we accumulate LSNs for either LNs or BINs.  The BIN
97  * accumulation mode provides an optimization for key-only traversals and for
98  * all traversals of duplicate DBs (in a dup DB, the data is included in the
99  * key).  In these cases we never need to fetch the LN, so we can sort and
100  * fetch the BIN LSNs instead.  This supports at least some types of traversals
101  * that are efficient when all BINs are not in the JE cache.
102  *
103  * We must only accumulate LN or BIN LSNs, never both, and never the LSNs of
104  * other INs (above level 1).  If we broke this rule, there would be no way to
105  * constrain memory usage in our non-recursive approach, since we could not
106  * easily predict in advance how much memory would be needed to fetch the
107  * nested nodes.  Even if we were able predict the memory needed, it would be
108  * of little advantage to sort and fetch a small number of higher level nodes,
109  * only to accumulate the LSNs of their descendants (which are much larger in
110  * number).  The much smaller number of higher level nodes would likely be
111  * fetched via random IO anyway, in a large data set anyway.
112  *
113  * The above justification also applies to the algorithm we use in LN mode, in
114  * which we accumulate and fetch only LN LSNs.  In this mode we always fetch
115  * BINs explicitly (not in LSN sorted order), if they are not resident, for the
116  * reasons stated above.
117  *
118  * Furthermore, in BIN mode we must account for BIN-deltas.  Phase I must keep
119  * a copy any BIN-deltas encountered in the cache.  And phase II must make two
120  * passes for the accumulated LSNs: one pass to load the deltas and another to
121  * load the full BINs and merge the previously loaded deltas.  Unfortunately
122  * we must budget memory for the deltas during phase I; since most BIN LSNs are
123  * for deltas, not full BINs, we assume that we will need to temporarily save a
124  * delta for each LSN.  This two pass approach is not like the recursive
125  * algorithm we rejected above, however, in two respects: 1) we know in advance
126  * (roughly anyway) how much memory we will need for both passes, and 2) the
127  * number of LSNs fetched in each pass is roughly the same.
128  *
129  * Data Lag
130  * --------
131  * In phase I, as an exception to what was said above, we sometimes process
132  * nodes that are resident in the Btree (in the JE cache) if this is possible
133  * without blocking.  The primary intention of this is to provide more recent
134  * data to the callback.  When accumulating BINs, if the BIN is dirty then
135  * fetching its LSN later means that some recently written LNs will not be
136  * included.  Therefore, if the callback would not block, we process the keys
137  * in a dirty BIN during phase I.  Likewise, when accumulating LNs in a
138  * deferred-write database, we process dirty LNs if the callback would not
139  * block.  When accumulating LN LSNs for a non-deferred-write database, we can
140  * go further and process all resident LNs, as long as the callback would not
141  * block, since we know that no LNs are dirty.
142  *
143  * In spite of our attempt to process resident nodes, we may not be able to
144  * process all of them if doing so would cause the callback to block.  When we
145  * can't process a dirty, resident node, the information added (new, deleted or
146  * updated records) since the node was last flushed will not be visible to the
147  * callback.
148  *
149  * In other words, the data presented to the callback may lag back to the time
150  * of the last checkpoint.  It cannot lag further back than the last
151  * checkpoint, because: 1) the scan doesn't accumulate LSNs any higher than the
152  * BIN level, and 2) checkpoints flush all dirty BINs.  For a DOS, the user may
153  * decrease the likelihood of stale data by increasing the DOS queue size,
154  * decreasing the LSN batch size, decreasing the memory limit, or performing a
155  * checkpoint immediately before the start of the scan.  Even so, it may be
156  * impossible to guarantee that all records written at the start of the scan
157  * are visible to the callback.
158  */

231      * WeakBinRefs are used to reduce the memory consumption for BIN deltas
232      * that are found in the je cache during phase 1. In an older implementation
233      * of DOS all such deltas were copied locally and the copies retained
234      * until phase 2b, when they would be merged with their associated full
235      * bins. Obviously this could consume a lot of memory.
236      *
237      * In the current implementation, dirty deltas are still treated the old
238      * way, i.e. copied (see below for a justification). But for clean deltas,
239      * a WeakBinRef is created when, during phase 1, such a delta is found in
240      * the je cache (the WeakBinRef is created while the delta is latched). At
241      * creation time, the WeakBinRef stores (a) a WeakReference to the BIN obj,
242      * (b) the lsn found in the parent slot for this bin (the on-disk image
243      * pointed to by this lsn must be the same as the latched, in-memory image,
244      * given that we use WeakBinRefs for clean deltas only) (c) the lsn of
245      * the full bin associated with this bin delta (copied out from the delta
246      * itself), and (d) the average size of deltas for the database and log
247      * file that the delta belongs to.
248      *
249      * DiskOrderedScanner.binDeltas is an ArrayList of Object refs, pointing
250      * to either WeakBinRefs (for clean deltas) or BINs (for copies of dirty
251      * deltas). The list is built during phase 1 and is used during phase 2b
252      * to merge the referenced deltas with their associated full BINs. The
253      * list is first sorted by full-bin LSN so that the full bins are fetched
254      * in disk order.
255      *
256      * Shortly after a WeakBinRef is created, its associated bin gets unlatched
257      * and remains unlatched until the WeakBinRef is used in phase 2b. As a
258      * result, anything can happen to the bin in between. The bin may be evicted
259      * and then garbage-collected, or it may be converted to a full BIN, split,
260      * logged, and mutated to a delta again. If the BIN obj gets GC-ed,
261      * this.get() will be null when we process this WeakBinRef in phase 2b.
262      * In this case, to avoid doing one or two random I/Os in the middle of
263      * sequential I/Os, this.binLsn is saved in a set of "deferred LSNs"
264      * (see below) to be processed in a subsequent iteration. If the BIN obj
265      * is still accessible, we compare its current full-bin LSN with the one
266      * saved in this.fullBinLsn. If they are different, we again add
267      * this.binLsn to deferred-LSNs set. Again this is done to avoid
268      * disturbing the sequential I/O (we could use the current full-bin LSN to
269      * merge the delta with the correct full bin, but this would require a
270      * random I/O), but also because the bin could have split, and there is no
271      * easy way to find the other bin(s) where slots from this bin were moved
272      * to. Note that detecting splits via comparing the 2 full-bin LSNs is a
273      * conservative approach that relies on splits being logged immediately. 
274      *
275      * WeakReferences are used to ref cached clean deltas in order to avoid
276      * counting the size of the delta in the DOS budget or having to pin the
277      * bin for a long period of time. Of course, if this.binLsn needs to be
278      * deferred for the reasons mentioned above, then an average delta size will
279      * be added to the memory usage for the DOS as well as the je cache.
280      *
281      * What about dirty cached deltas? We could be doing the same as for clean
282      * deltas. However, for dirty deltas, their unsaved updates could have
283      * happened before the start of the DOS and we would loose these updates
284      * if instead of processing the original bin delta, we have to fetch
285      * this.binLsn from disk. So, we would be violating the DOS contract that
286      * says that the returned data are not older than the start of the DOS.

724     private void scanSerial() {
725 
726         int dbidx = 0;
727         boolean overBudget;
728         DBContext ctx = null;
729 
730         while (true) {
731 
732             /*
733              * Phase I.
734              */
735             try {
736                 /*
737                  * Skip phase 1 if we have already exceeded the DOS budget
738                  * due to delta lsns deferred from phase 2b.
739                  */
740                 overBudget = accLimitExceeded();
741 
742                 while (dbidx < numDBs && !overBudget) {
743 
744                     ctx = dbs[dbidx];
745 
746                     if (ctx.parent == null) {
747                         getFirstIN(ctx, ctx.prevEndingKey);
748                     }
749 
750                     if (ctx.done) {
751                         ++dbidx;
752                         assert(ctx.parent == null);
753                         continue;
754                     }
755 
756                     while (ctx.parent != null) {
757 
758                         if (binsOnly) {
759                             accumulateBINs(ctx);
760                         } else {
761                             accumulateLNs(ctx);
762                         }
763 
764                         if (accLimitExceeded()) {
765                             overBudget = true;
766                             break;
767                         }
768 
769                         processor.checkShutdown();
770 
771                         getNextIN(ctx);
772                     
773                         if (ctx.done) {
774                             ++dbidx;
775                             assert(ctx.parent == null);
776                         }
777                     }
778                 }
779 
780             } finally {
781                 if (ctx.parent != null && ctx.parentIsLatched) {
782                     ctx.parent.releaseLatchIfOwner();
783                 }
784             }
785 
786             /*
787              * Phase II.
788              */
789             if (binsOnly) {
790                 fetchAndProcessBINs();
791             } else {
792                 fetchAndProcessLNs();
793             }
794 
795             /*
796              * Check if DOS is done; if not prepare for next iteration
797              */
798             if (dbidx >= numDBs && (!binsOnly || deferredLsns.isEmpty())) {
799                 break;
800             }

803         }

937

1016     /**
1017      * Implements guts of phase I in binsOnly mode.  Accumulates BIN deltas and
1018      * BIN LSNs for the children of the given level 2 IN parent, and processes
1019      * resident BINs under certain conditions; see algorithm at top of file.
1020      */
1021     private void accumulateBINs(DBContext ctx) {
1022 
1023         OffHeapCache ohCache = env.getOffHeapCache();
1024 
1025         while (ctx.pidx < ctx.parent.getNEntries()) {
1026 
1027             /* Skip BINs that were processed on the previous iteration. */
1028             if (skipParentSlot(ctx)) {
1029                 ++ctx.pidx;
1030                 continue;
1031             }
1032 
1033             /*
1034              * A cached delta must be copied it if it may contain any keys <=
1035              * prevEndingKey. Otherwise, if it is the last bin processed in
1036              * the previous iteration, it will be processed again if we take
1037              * a weak ref to the delta and this weak ref is then cleared and
1038              * the delta lsn being deferred as a result. Because no key checking
1039              * is done for deferred bins, processing the bin again would result
1040              * in duplicate records being returned to the app.
1041              */
1042             if (ctx.prevEndingKey == null ||
1043                 (!ctx.safeToUseCachedDelta &&
1044                  ctx.pidx > 0 &&
1045                  Key.compareKeys(ctx.prevEndingKey,
1046                                  ctx.parent.getKey(ctx.pidx),
1047                                  ctx.dbImpl.getKeyComparator()) < 0)) {
1048 
1049                 ctx.safeToUseCachedDelta = true;
1050             }
1051 
1052             boolean waitForConsumer = false;
1053             int binNEntries = 0;
1054             long binLsn = ctx.parent.getLsn(ctx.pidx);
1055             int ohBinId = ctx.parent.getOffHeapBINId(ctx.pidx);
1056             boolean ohBinPri2 = ctx.parent.isOffHeapBINPri2(ctx.pidx);
1057 
1058             ctx.lastBinLsn = binLsn;
1059 
1060             BIN bin = (BIN)ctx.parent.getTarget(ctx.pidx);
1061 
1062             if (bin != null) {
1063                 bin.latch(CacheMode.UNCHANGED);
1064             }
1065 
1066             try {
1067                 if (bin != null || ohBinId >= 0) {
1068 
1069                     boolean isBinDelta;
1070                     OffHeapCache.BINInfo ohInfo = null;
1071 
1072                     if (bin != null) {
1073                         isBinDelta = bin.isBINDelta();
1074                     } else {
1075                         ohInfo = ohCache.getBINInfo(env, ohBinId);
1076                         isBinDelta = ohInfo.isBINDelta;
1077                     }
1078 
1079                     if (isBinDelta) {
1080 
1081                         if (bin != null) {
1082                             if (bin.getDirty() || !ctx.safeToUseCachedDelta) {
1083                                 addDirtyDeltaRef(bin.cloneBINDelta());
1084                             } else {
1085                                 addCleanDeltaRef(ctx, binLsn, bin);
1086                             }
1087 
1088                         } else {
1089                             if (ctx.parent.isOffHeapBINDirty(ctx.pidx) ||
1090                                 !ctx.safeToUseCachedDelta) {
1091 
1092                                 addDirtyDeltaRef(
1093                                     ohCache.loadBIN(env, ohBinId));
1094                             } else {
1095                                 addCleanDeltaOffHeapRef(
1096                                     ctx, binLsn, ohInfo.fullBINLsn,
1097                                     ohBinId, ohBinPri2);
1098                             }
1099                         }
1100 
1101                         ++ctx.pidx;
1102                         if (scanSerial) {
1103                             continue;
1104                         } else {
1105                             return;
1106                         }
1107                     }
1108 
1109                     if (bin == null) {
1110                         bin = ohCache.loadBIN(env, ohBinId);
1111                         bin.latchNoUpdateLRU(ctx.dbImpl);
1112                     }
1113 
1114                     binNEntries = bin.getNEntries();
1115                     if (binNEntries == 0) {
1116                         ++ctx.pidx;
1117                         continue;
1118                     }
1119 
1120                     /*
1121                      * Skip the bin if its last key is <= prevEndingKey.
1122                      * Normally, this should happen only if the bin is the last
1123                      * bin from the previous iteration and the 1st bin of the
1124                      * current iteration.
1125                      */
1126                     if (ctx.prevEndingKey != null) {
1127                         int cmp = Key.compareKeys(
1128                             bin.getKey(binNEntries - 1),
1129                             ctx.prevEndingKey,
1130                             ctx.dbImpl.getKeyComparator());
1131 
1132                         if (cmp <= 0) {
1133                             ++ctx.pidx;
1134                             continue;
1135                         }
1136                     }
1137                 }
1138 
1139                 /*
1140                  * If the BIN is not resident, accumulate this LSN for later
1141                  * processing during phase 2. During phase 2a, if the lsn
1142                  * points to a delta, that delta will be fetched from disk and
1143                  * stored until all lsns accumulated here have been fetched.
1144                  * Then, in phase 2b the full BINs corresponding to the stored
1145                  * deltas will be read in lsn order. So, we must budget memory
1146                  * for these deltas. Since most lsns point to deltas, we assume
1147                  * that they all point to deltas. 
1148                  */
1149                 if (bin == null || processor.getCapacity() < binNEntries) {
1150 
1151                     lsnAcc.add(binLsn);
1152                     ++numLsns;
1153 
1154                     addLocalMemory(8 + getDeltaMemSize(ctx, binLsn));
1155 
1156                     if (debug) {
1157                         System.out.println(
1158                             "Phase 1." + nIterations +
1159                             ": accumulated bin lsn: " + binLsn);
1160                     }
1161 
1162                 } else if (processor.canProcessWithoutBlocking(binNEntries)) {
1163 
1164                     if (debug) {
1165                         System.out.println(
1166                             "Phase 1." + nIterations +
1167                             ": Processing bin: " + bin.getNodeId());
1168                     }
1169 
1170                     processBINInternal(ctx, bin, false);
1171 
1172                 } else {
1173                     if (debug) {
1174                         System.out.println(
1175                             "Phase 1." + nIterations +
1176                             ": Producer must wait before it can process bin " +
1177                             bin.getNodeId());
1178                     }
1179 
1180                     waitForConsumer = true;
1181                     ctx.binKey = bin.getKey(0);
1182                 }
1183 
1184             } finally {
1185                 if (bin != null) {
1186                     bin.releaseLatch();
1187                 }
1188             }
1189 
1190             if (waitForConsumer) {
1191                 waitForConsumer(ctx, binNEntries);
1192             } else {
1193                 ++ctx.pidx;
1194                 if (!scanSerial) {
1195                     return;
1196                 }
1197             }
1198         } /* parent slot iteration */
1199     }

1209          * WeakBinRef obj, plus the ref to it in this.binDeltas. For
1210          * the local memory we count 2 additional overheads: (a) another
1211          * ref in the deltaArray allocated at the start of phase 2a and

1216                        2 * SIZEOF_JAVA_REF +
