23 /**
24  * Accumulates changes to the utilization profile locally in a single thread.
25  *
26  * <p>Per-database information is keyed by DatabaseImpl so that no tree lookup
27  * of a database is required (as when a DatabaseId is used).</p>
28  *
29  * <p>The countNewLogEntry, countObsoleteNode and countObsoleteNodeInexact
30  * methods may be called without taking the log write latch.  Totals and offset
31  * are accumulated locally in this object only, not in DatabaseImpl
32  * objects.</p>
33  *
34  * <p>When finished with this object, its information should be added to the
35  * Environment's UtilizationTracker and DatabaseImpl objects by calling
36  * transferToUtilizationTracker under the log write latch.  This is done in the
37  * Checkpointer, Evictor and INCompressor by calling
38  * UtilizationProfile.flushLocalTracker which calls
39  * LogManager.transferToUtilizationTracker which calls
40  * BaseLocalUtilizationTracker.transferToUtilizationTracker.</p>
41  */
