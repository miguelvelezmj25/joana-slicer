18 /**
19  * The common base class for all exceptions that result from record lock
20  * conflicts during read and write operations.
21  *
22  * <p>This exception normally indicates that a transaction may be retried.
23  * Catching this exception, rather than its subclasses, is convenient and
24  * recommended for handling lock conflicts and performing transaction retries
25  * in a general purpose manner.  See below for information on performing
26  * transaction retries.</p>
27  *
28  * <p>The exception carries two arrays of transaction ids, one of the owners and
29  * the other of the waiters, at the time of the lock conflict.  This
30  * information may be used along with the {@link Transaction#getId Transaction
31  * ID} for diagnosing locking problems. See {@link #getOwnerTxnIds} and {@link
32  * #getWaiterTxnIds}.</p>
33  *
34  * <p>The {@link Transaction} handle is invalidated as a result of this
35  * exception.</p>
36  *
37  * <a name="retries"><h3>Performing Transaction Retries</h3></a>
38  *
39  * <p>If a lock conflict occurs during a transaction, the transaction may be
40  * retried by performing the following steps.  Some applications may also wish
41  * to sleep for a short interval before retrying, to give other concurrent
42  * transactions a chance to finish and release their locks.</p>
43  * <ol>
44  * <li>Close all cursors opened under the transaction.</li>
45  * <li>Abort the transaction.</li>
46  * <li>Begin a new transaction and repeat the operations.</li>
47  * </ol>
48  *
49  * <p>To handle {@link LockConflictException} reliably for all types of JE
50  * applications including JE-HA applications, it is important to handle it when
51  * it is thrown by all {@link Database} and {@link Cursor} read and write
52  * operations.</p>
53  *
54  * <p>The following example code illustrates the recommended approach.  Note
55  * that the {@code Environment.beginTransaction} and {@code Transaction.commit}
56  * calls are intentially inside the {@code try} block.  When using JE-HA, this
57  * will make it easy to add a {@code catch} for other exceptions that can be
58  * resolved by retrying the transaction, such as consistency exceptions.</p>
59  *
60  * <pre class="code">
61  *  void doTransaction(final Environment env,
62  *                     final Database db1,
63  *                     final Database db2,
64  *                     final int maxTries)
65  *      throws DatabaseException {
66  *
67  *      boolean success = false;
68  *      long sleepMillis = 0;
69  *      for (int i = 0; i &lt; maxTries; i++) {
70  *          // Sleep before retrying.
71  *          if (sleepMillis != 0) {
72  *              Thread.sleep(sleepMillis);
73  *              sleepMillis = 0;
74  *          }
75  *          Transaction txn = null;
76  *          try {
77  *              txn = env.beginTransaction(null, null);
78  *              final Cursor cursor1 = db1.openCursor(txn, null);
79  *              try {
80  *                  final Cursor cursor2 = db2.openCursor(txn, null);
81  *                  try {
82  *                      // INSERT APP-SPECIFIC CODE HERE:
83  *                      // Perform read and write operations.
84  *                  } finally {
85  *                      cursor2.close();
86  *                  }
87  *              } finally {
88  *                  cursor1.close();
89  *              }
90  *              txn.commit();
91  *              success = true;
92  *              return;
93  *          } catch (LockConflictException e) {
94  *              sleepMillis = LOCK_CONFLICT_RETRY_SEC * 1000;
95  *              continue;
96  *          } finally {
97  *              if (!success) {
98  *                  if (txn != null) {
99  *                      txn.abort();
100  *                  }
101  *              }
102  *          }
103  *      }
104  *      // INSERT APP-SPECIFIC CODE HERE:
105  *      // Transaction failed, despite retries.
106  *      // Take some app-specific course of action.
107  *  }</pre>
108  *
109  * <p>For more information on transactions and lock conflicts, see <a
110  * href="{@docRoot}/../TransactionGettingStarted/index.html"
111  * target="_top">Writing Transactional Applications</a>.</p>
112  *
113  * @since 4.0
114  */
