367     /**
368      * Convenience logging method.  See logInternal.
369      *
370      * For a deferred-write database, the logging will not actually occur and
371      * a transient LSN will be created and returned if the currLsn is NULL;
372      * otherwise the currLsn is returned. 
373      *
374      * However, if the embeddedness of the LN changes, we must create a real
375      * logrec for the op. The following scenario is an example of what can
376      * go wrong if we don't log embeddedness changes:
377      *
378      * - R1 exists before DB is opened in DW mode.
379      * - R1 is embedded, so its on-disk image has been counted obsolete and
380      *   might have been deleted already.
381      * - R1 is updated in DW mode, and its new version, R2, is not embedded
382      *   (and not logged). Furthermore, the slot LSN points to R1 still.
383      * - R2 gets evicted and logged. As a result, R1 will be counted as
384      *   obsolete again, because the slot points to R1 and says R1 is not
385      *   embedded. 
386      *
387      * If we do log embeddedness changes, then in the above scenario, R2 will
388      * be logged and this logging will not count R1 as obsolete, because the
389      * embedded flag in the slot still says that R1 is embedded. After R2 is
390      * logged, the slot is updated (both the slot LSN and the embedded flag).
391      *
392      * In general, the embedded flag in the slot must be in accord with the
393      * embedded flag in the last logged version of a record, even if multiple
394      * updates were done on the record since the last time it was logged and
395      * the current logging event. 
396      */

456      * Generate and write to the log a logrec describing an operation O that
457      * is being performed on a record R with key K. O may be an insertion,
458      * update, deletion, migration, or in the case of a DW DB, an eviction
459      * or checkpoint of a dirty LN.
460      *
461      * Let T be the locker performing O. T is null in case of DW eviction/ckpt.
462      * Otherwise, T holds a lock on R and it will keep that lock until it
463      * terminates. In case of a CUD op, the lock is an exclusive one; in 
464      * case of LN migration, it's a shared one (and T is non-transactional).
465      *
466      * - Let Rc be the current version of R (before O). The absence of R from
467      *   the DB is considered as a special "deleted" version. Rc may be the
468      *   deleted version.
469      * - If T is a Txn, let Ra be the version of R before T write-locked R. Ra
470      *   may be the deleted version. Ra and Rc will be the same if O is the
471      *   very 1st op on R by T.
472      * - Let Rn be R's new version (after O). Rc and Rn will be the same if O
473      *   is migration or DW eviction/ckpt.
474      *
475      * - Let Ln be the LSN of the logrec that will be generated here to
476      *   describe O.
477      * - Let Lc be the current LSN value in R's slot, or NULL if no such slot
478      *   exists currently. If an R slot exists, then for a non-DW DB, Lc points
479      *   to Rc, or may be NULL if Rc is the deleted version. But for a DW DB,
480      *   Lc may point to an older version than Rc, or it may be transient.
481      * - If T is a Txn, let La be the LSN value in R's slot at the time T 
482      *   write-locked R, or NULL if no such slot existed at that time. 
483      *
484      * @param isInsertion Whether this CUDop is an insertion (possibly with
485      * slot reuse.
486      *
487      * @param locker The locker T. If non-null, a write lock will be acquired
488      * by T on Ln's LSN.
489      *
490      * WARNING: Be sure to pass null for the locker param if the new LSN should
491      * not be locked.
492      *
493      * @param writeLockInfo It is non-null if and only if T is a Txn. It
494      * contains info that must be included in Ln to make it undoable if T
495      * aborts. Specifically, it contains:
496      *
497      * - abortKD   : True if Ra is the deleted version; false otherwise.
498      * - abortLSN  : The La LSN as defined above.
499      * - abortKey  : The key of Ra, if Ra was embedded in the parent BIN and
500      *               the containing DB allows key updates.
501      * - abortData : The data of Ra, if Ra was embedded in the parent BIN.
502      *
503      * When the new LSN is write-locked, a new WriteLockInfo is created and
504      * the above info is copied into it. Normally this parameter should be
505      * obtained from the prepareForInsert or prepareForUpdate method of
506      * CursorImpl.LockStanding.
507      *
508      * @param newEmbeddedLN Whether Rn will be embedded into the parent BIN.
509      * If true, Ln will be counted as an "immediately obsolete" logrec.
510      *
511      * @param newKey Rn's key. Note: Rn's data is not passed as a parameter to
512      * this method because it is stored in this LN. Rn (key and data) will be
513      * stored in Ln. Rn's key will also be stored in the parent BIN, and if
514      * newEmbeddedLN is true, Rn's data too will be stored there.
515      *
516      * @param newExpiration the new expiration time in days or hours.
517      *
518      * @param newExpirationInHours whether the new expiration time is in hours.
519      *
520      * @param currEmbeddedLN Whether Rc's data is embedded into the parent
521      * BIN. If true, Lc has already been counted obsolete.
522      *
523      * @param currLsn The Lc LSN as defined above. Is given as a param to this
524      * method to count the associated logrec as obsolete (which must done under
525      * the LWL), if it has not been counted already.
526      *
527      * @param currSize The size of Lc (needed for obsolete counting).
528      *
529      * @param isInsertion True if the operation is an insertion (including
530      * slot reuse). False otherwise.
531      */
532     private LogItem logInternal(
533         final EnvironmentImpl envImpl,
534         final DatabaseImpl dbImpl,
535         final Locker locker,
536         final WriteLockInfo writeLockInfo,
537         final boolean newEmbeddedLN,
538         final byte[] newKey,
539         final int newExpiration,
540         final boolean newExpirationInHours,
541         final boolean currEmbeddedLN,
542         final long currLsn,
543         final int currSize,
544         final boolean isInsertion,
545         final boolean backgroundIO,
546         final ReplicationContext repContext)
547         throws DatabaseException {
548 
549         assert(getClass() == LN.class ||
550                getClass() == VersionedLN.class ||
551                !newEmbeddedLN);
552 
553         if (envImpl.isReadOnly()) {
554             /* Returning a NULL_LSN will not allow locking. */
555             throw EnvironmentFailureException.unexpectedState(
556                 "Cannot log LNs in read-only env.");
557         }
558 
559         /*
560          * Check that a replicated txn is used for writing to a replicated DB,
561          * and a non-replicated locker is used for writing to a
562          * non-replicated DB. This is critical for avoiding corruption when HA
563          * failover occurs [#23234] [#23330].
564          *
565          * Two cases are exempt from this rule:
566          *
567          *  - The locker is null only when performing internal logging (not a
568          *    user operation), such as cleaner migration and deferred-write
569          *    logging.  This is always non-transactional and non-replicated, so
570          *    we can skip this check.  Note that the cleaner may migrate an LN
571          *    in a replicated DB, but this is not part ot the rep stream.
572          *
573          *  - Only NameLNs that identify replicated DBs are replicated, not
574          *    all NameLNs in the naming DB, so the naming DB is exempt.
575          *
576          * This guard should never fire because of two checks made prior to
577          * logging:
578          *
579          *  - When a user txn in a replicated environment is not configured for
580          *    local-write and a write operation is attempted (or when the
581          *    opposite is true), the Cursor class will throw
582          *    UnsupportedOperationException. See Locker.isLocalWrite.
583          *
584          *  - On a replica, writes to replicated DBs are disallowed even when
585          *    local-write is false.  This is enforced by the ReadonlyTxn class
586          *    which throws ReplicaWriteException in this case.
587          */
588         final boolean isNamingDB = dbImpl.getId().equals(DbTree.NAME_DB_ID);
589 
590         if (!isNamingDB &&
591             envImpl.isReplicated() &&
592             locker != null &&
593             dbImpl.isReplicated() != locker.isReplicated()) {
594 
595             throw EnvironmentFailureException.unexpectedState(
596                 (locker.isReplicated() ?
597                     "Rep txn used to write to non-rep DB" :
598                     "Non-rep txn used to write to rep DB") +
599                 ", class = " + locker.getClass().getName() +
600                 ", txnId = " + locker.getId() +
601                 ", dbName = " + dbImpl.getDebugName());
602         }
603 
604         /*
605          * As an additional safeguard, check that a replicated txn is used when
606          * the operation is part of the rep stream, and that the inverse is
607          * also true. The naming DB is exempt for the same reason as above.
608          */
609         if (!isNamingDB) {
610 
611             boolean isRepLocker = (locker != null) && locker.isReplicated();
612 
613             if (repContext.inReplicationStream() != isRepLocker) {
614                 throw EnvironmentFailureException.unexpectedState(
615                     (isRepLocker ?
616                         "Rep txn used to write outside of rep stream" :
617                         "Non-rep txn used to write in rep stream") +
618                     ((locker != null) ?
619                         (", class = " + locker.getClass().getName() +
620                          ", txnId = " + locker.getId()) :
621                         ", null locker") +
622                     ", dbName = " + dbImpl.getDebugName());
623             }
624         }
625 
626         LogEntryType entryType;
627         Txn txn = null;
628         long abortLsn = DbLsn.NULL_LSN;
629         boolean abortKD = false;
630         byte[] abortKey = null;
631         byte[] abortData = null;
632         long abortVLSN = VLSN.NULL_VLSN_SEQUENCE;
633         int abortExpiration = 0;

670          * temporary DBs are removed during recovery anyway.

699         if (txn != null && currLsn == abortLsn) {

710                  * ensure that the view we get is consistent. [#17204]

714                 }

735              * the Btree.  This avoids incorrect assertions in other threads.
736              */

742          * Lock non-blocking, since no contention is possible on the new LSN.
743          * If the locker is transactional, a new WriteLockInfo is created for
744          * the new LSN and stored in the locker. lockResult points to that
745          * WriteLockInfo. Since this new WriteLockInfo and the WriteLockInfo
746          * given as input to this method refer to the same logical record,
747          * the info from the given WriteLockInfo is copied to the new one.
748          */
749         if (locker != null) {
750             final long newLsn = item.lsn;
751 
752             final LockResult lockResult = locker.nonBlockingLock(
753                 newLsn, LockType.WRITE, false /*jumpAheadOfWaiters*/, dbImpl);
754 
755             assert lockResult.getLockGrant() != LockGrantType.DENIED :
756                    DbLsn.getNoFormatString(newLsn);
757 
758             lockResult.copyWriteLockInfo(writeLockInfo);
759         }
760 
761         /* In a dup DB, do not expect embedded LNs or non-empty data. */
762         if (dbImpl.getSortedDuplicates() &&
763             (newEmbeddedLN || (data != null && data.length > 0))) {
764 
765             throw EnvironmentFailureException.unexpectedState(
766                 envImpl,
767                 "[#25288] emb=" + newEmbeddedLN +
768                 " key=" + Key.getNoFormatString(newKey) +
769                 " data=" + Key.getNoFormatString(data) +
770                 " vlsn=" + item.header.getVLSN() +
771                 " lsn=" + DbLsn.getNoFormatString(currLsn));
772         }
773 
774         return item;

1137         final int doff = entry.getPartialOffset();

1144             newData = LogUtils.ZERO_LENGTH_BYTE_ARRAY;
1145         } else {

1152         if (slicelen > 0) {
1153             System.arraycopy(foundDataBytes, 0, newData, pos, slicelen);
1154         }

1158         slicelen = entry.getSize();
1159         System.arraycopy(entry.getData(), entry.getOffset(), newData, pos,
1160                          slicelen);
1161         pos += slicelen;
1162

1164         slicelen = origlen - (doff + dlen);
1165         if (slicelen > 0) {
1166             System.arraycopy(foundDataBytes, doff + dlen, newData, pos,
