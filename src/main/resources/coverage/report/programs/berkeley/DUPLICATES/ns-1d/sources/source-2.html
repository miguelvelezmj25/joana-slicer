38 /**
39  * The GlobalCBVLSN was used prior to JE 7.5 for ensuring that nodes do not
40  * delete files needed for feeding other nodes. The methods in this class are
41  * disabled if the GlobalCBVLSN is defunct -- see below.
42  *
43  * <p>In version 7.5 and greater we instead retain all files up to the disk
44  * limits just in case they are needed for replication, and the GlobalCBVLSN
45  * is no longer needed. The {@link com.sleepycat.je.cleaner.FileProtector} is
46  * used for preventing file deletion on a local node. See [#25220] in the
47  * JE 7.5 change log for a list of related external changes.</p>
48  *
49  * <p>The global and local CBVLSNs are completely defunct (no longer
50  * maintained) in a rep group where all nodes are running JE 7.5 or greater.
51  * In a mixed version group of new (7.5 or later) and old (pre-7.5) nodes,
52  * we must continue to maintain the GlobalCBVLSN for the sake of the old
53  * nodes. If we did not do this, for example, the following scenario could
54  * occur:
55  * <pre>
56  *  - RF-3, N1 is new, N2 and N3 are old.
57  *  - N1 is the master, N2 is an up-to-date replica, N3 is a lagging replica.
58  *  - N2 deletes files that would be needed to feed N3, because the GlobalVLSN
59  *    is not updated and therefore N2 doesn't know that N3 is lagging.
60  *  - N1 goes down, N2 is elected master
61  *  - N3 requires a network restore because N2 doesn't have the files that N3
62  *    needs for syncup.
63  *</pre>
64  *
65  * <p>The c.s.je.rep.impl.node.cbvlsn package contains most of the code for
66  * maintaining the CBVLSN. This code is isolated here for two reasons:</p>
67  * <ul>
68  *     <li>To make it clear that it is not in play for groups of all new nodes.
69  *     We don't have to think about the CBVLSN at all in this case.</li>
70  *     <li>To make it easier to eventually remove this code entirely, when
71  *     backward compatibility with JE 7.4 and earlier is dropped.</li>
72  *</ul>
73  *
74  * <p>When it is not yet defunct, the GlobalCBVLSN class represents this
75  * node's view of the global CBVLSN. Each node has its own view of the global
76  * CBVLSN, based upon its local replicated copy of the rep group db. There is
77  * a single instance of the GlobalCBVLSN and it exists for the lifetime of the
78  * RepNode.</p>
79  *
80  * <p>A global CBVLSN is a per-environment value, and is safeguarded from
81  * decreasing during the lifetime of a single RepImpl. Because nodes crash and
82  * re-sync, and new nodes join, it's possible that the persisted local cbvlsns
83  * can decrease, and drag the global cbvlsn value down, but those decreases
84  * are ignored during the lifetime of the global CBVLSN instance.</p>
85  *
86  * <p>On pre-7.5 nodes, the global CBVLSN is used by:</p>
87  * <ul>
88  *     <li>As a barrier to file deletion by the Cleaner.</li>
89  *     <li>The Feeder which only serves log records in the interval:
90  *     [GlobalCBVLSN .. VLSNRange.last]</li>
91  *     <li>The Replica which uses the interval [GlobalCBVLSN .. VLSNRange.last]
92  *     at syncup time.</li>
93  *</ul>
94  */

240     /**
241      * Recalculates the GlobalVLSN when it is not defunct. When it is defunct,
242      * does nothing.
243      *
244      * <p>For sake of old nodes, the globalCBVLSN is computed as it was
245      * earlier: the minimum of CBVLSNs after discarding CBVLSNs that are
246      * obsolete. A CBVLSN is considered obsolete if it has not been updated
247      * within a configurable time interval relative to the time that the
248      * most recent CBVLSN was updated.</p>
249      *
250      * <p>Note that the read of GroupInfo is not protected, and that groupInfo
251      * could be changing. That's okay, because we guarantee that none of the
252      * local CBVLSNs can be LT globalCBVLSN. If a local CBVLSN is written, and
253      * we miss it, it only means that this recalculation of global CBVLSN is
254      * too pessimistic -- it's too low.</p>
255      *
256      * <p>Secondary nodes do not appear in the RepGroupDB, but the feeder has
257      * local CBVLSN values for them which are part of this calculation.
258      * Secondary nodes and new nodes have their VLSN ranges protected by the
259      * mechanism which tries to ensure that files which may be needed by active
260      * feeders are not candidates for deletion.</p>
261      */
262     public void recalculate(RepGroupImpl groupInfo) {
263 
264         if (defunct) {
265             return;
266         }
267 
268         /* Find the time the highest CBVLSN was computed. */
269         VLSN maxCBVLSN = NULL_VLSN;
270         long latestBarrierTime = 0;
271         String nodeName = null;
272         for (RepNodeImpl node : groupInfo.getDataMembers()) {
273 
274             CleanerBarrierState nodeBarrier = node.getBarrierState();
275             VLSN cbvlsn = nodeBarrier.getLastCBVLSN();
276 
277             if ((cbvlsn == null) || cbvlsn.isNull()) {
278                 continue;
279             }
280 
281             /*
282              * Count all nodes when finding the max time, including old nodes
283              * that are in the middle of syncup and have not established their
284              * low point .
285              */
286             final long nodeBarrierTime = nodeBarrier.getBarrierTime();
287 
288             if (maxCBVLSN.compareTo(cbvlsn) <= 0) {
289 
290                 /*
291                  * Use min, since it represents the real change when they are
292                  * equal.
293                  */
294                 latestBarrierTime = cbvlsn.equals(maxCBVLSN) ?
295                     Math.min(nodeBarrierTime, latestBarrierTime) :
296                     nodeBarrierTime;
297                 maxCBVLSN = cbvlsn;
298 
299                 /*
300                  * Track the name of the node holding the maximum CBVLSN, since
301                  * that node pins that VLSN
302                  */
303                 nodeName = node.getName();
304             }
305         }
306 
307         if (latestBarrierTime == 0) {
308             /* No cbvlsns entered yet, don't bother to recalculate. */
309             return;
310         }
311 
312         if (maxCBVLSN.isNull()) {
313             /* No cbvlsns entered yet, don't bother to recalculate. */
314             return;
315         }
316 
317         /*
318          * Now find the min CBVLSN that has not been timed out. This may mean
319          * that the min CBVLSN == NULL_VLSN, for old nodes that have not yet
320          * finished syncup.
321          */
322         VLSN newGroupCBVLSN = maxCBVLSN;
323         long nodeBarrierTime = 0;
324         for (RepNodeImpl node : groupInfo.getDataMembers()) {
325 
326             CleanerBarrierState nodeBarrier = node.getBarrierState();
327             VLSN nodeCBVLSN = nodeBarrier.getLastCBVLSN();
328 
329             if ((nodeCBVLSN == null) || nodeCBVLSN.isNull()) {
330                 continue;
331             }
332 
333             if (((latestBarrierTime - nodeBarrier.getBarrierTime()) <=
334                  streamTimeoutMs) &&
335                 (newGroupCBVLSN.compareTo(nodeCBVLSN) > 0)) {
336                 newGroupCBVLSN = nodeCBVLSN;
337 
338                 /*
339                  * A node is pinning the CBVLSN because it is lagging and has
340                  * not timed out
341                  */
342                 nodeName = node.getName();
343                 nodeBarrierTime = nodeBarrier.getBarrierTime();
344             }
345         }
346 
347         /*
348          * Adjust to retain min number of VLSNs, while ensuring we stay within
349          * the current VLSN range.
350          */
351         newGroupCBVLSN =
352             new VLSN(newGroupCBVLSN.getSequence() - minRetainedVLSNs);
353 
354         final VLSNIndex vlsnIndex = repImpl.getVLSNIndex();
355         final VLSN rangeFirst = (vlsnIndex != null) ?
356             vlsnIndex.getRange().getFirst() : VLSN.FIRST_VLSN;
357 
358         /*
359          * Environments where the minRetainedVLSNs was expanded need to ensure
360          * the global cbvlsn still stays within the vlsn range.
361          */
362         if (rangeFirst.compareTo(newGroupCBVLSN) > 0) {
363             newGroupCBVLSN = rangeFirst;
364         }
365 
366         updateGroupCBVLSN(groupInfo, newGroupCBVLSN, nodeName,
367                           nodeBarrierTime, latestBarrierTime);
368     }

370     /*
371      * Update the group CBVLSN, but only if the newGroupCBVLSN is more recent
372      * This is to ensure that the group CBVLSN can only advance during the
373      * lifetime of this instance.
374      */
375     private void updateGroupCBVLSN(RepGroupImpl groupInfo,
376                                    VLSN newGroupCBVLSN,
377                                    String nodeName,
378                                    long nodeBarrierTime,
379                                    long latestBarrierTime) {
380         assert !defunct;
381         boolean changed = false;
382         String cbvlsnLoweredMessage = null;
383         VLSN oldCBVLSN = VLSN.NULL_VLSN;
384 
385         synchronized(this) {
386 
387             /*
388              * Be sure not to do anything expensive in this synchronized
389              * section, such as logging.
390              */
391             if (newGroupCBVLSN.compareTo(groupCBVLSN) > 0) {
392                 VLSNRange currentRange = repImpl.getVLSNIndex().getRange();
393                 if (!currentRange.contains(newGroupCBVLSN) &&
394                     logger.isLoggable(Level.FINE)) {
395                     cbvlsnLoweredMessage =
396                         "GroupCBVLSN: " + newGroupCBVLSN +
397                         " is outside VLSN range: " + currentRange +
398                         " Current group:" + groupInfo;
399                 } else {
400                     oldCBVLSN = groupCBVLSN;
401                     groupCBVLSN = newGroupCBVLSN;
402                     changed = true;
403                 }
404             }
405         }
406 
407         if (logger.isLoggable(Level.FINE)) {
408             if (cbvlsnLoweredMessage != null) {
409                 LoggerUtils.fine(logger, repImpl, cbvlsnLoweredMessage);
410             }
411 
412             if (changed) {
413                 LoggerUtils.fine(logger, repImpl,
414                                  "Global CBVLSN changed from " + oldCBVLSN +
415                                  " to " + newGroupCBVLSN);
416             }
417         }
418     }

420     /**
421      * Returns a VLSN appropriate for the RestoreResponse.cbvlsn field when the
422      * GlobalCBVLSN is not defunct. When it is defunct, returns a null VLSN.
423      *
424      * <p>When sending a RestoreResponse to a (potentially) old node, we
425      * supply a VLSN that will cause selection of reasonably current nodes as a
426      * server (feeder) for the network restore. The return value is the VLSN
427      * range end of this node (the master) minus a value that will allow
428      * up-to-date servers to qualify (NETWORKBACKUP_MAX_LAG). Old nodes
429      * will reject servers whose VLSN range does not cover this VLSN.</p>
430      *
431      * <p>In JE 7.4 and earlier, the "group CBVLSN" was used for this value.
432      * This was incorrect, because it was the lower bound for lagging replicas
433      * and reserved files. In JE 7.5 we improve on this by sending the
434      * value described above, when older nodes must be supported.</p>
435      *
436      * <p>When GlobalCBLVSN is defunct, a null VLSN is returned because the
437      * RestoreResponse.cbvlsn field is not used at all. See the updated
438      * 'Algorithm' in {@link NetworkRestore}.</p>
439      */
440     public VLSN getRestoreResponseVLSN(final VLSNRange range) {
441 
442         if (defunct) {
443             return VLSN.NULL_VLSN;
444         }
445 
446         final long vlsn = range.getLast().getSequence() -
447             repImpl.getConfigManager().getInt(RepParams.NETWORKBACKUP_MAX_LAG);
448 
449         return new VLSN(Math.max(0, vlsn));
450     }
