19 /**
20  * Thrown by the {@link Environment#close()} when the current process must be
21  * shut down and restarted before re-opening the Environment.
22  * <p>
23  * If during close(), a badly behaved internal thread cannot be stopped,
24  * then the JVM process must be stopped and restarted. The close() method first
25  * attempts a soft shutdown of each thread. If that fails to stop the thread,
26  * it is interrupted. If that fails to stop the thread, because it never
27  * becomes interruptible, then {@code EnvironmentWedgedException} is thrown by
28  * close(), after performing as much of the normal shutdown process as
29  * possible. Before this exception is thrown, a full thread dump is logged, to
30  * aid in debugging.
31  * <p>
32  * Note that prior to calling close(), if JE attempts to shut down an internal
33  * thread and it cannot be shut down, the Environment will be {@link
34  * Environment#isValid() invalidated}, also causing an {@code
35  * EnvironmentWedgedException} to be thrown. In this case (as in all other
36  * cases where an {@link EnvironmentFailureException} is thrown and the
37  * Environment is invalidated), the application should call {@link
38  * Environment#close()}. The close() method will throw {@code
39  * EnvironmentWedgedException} in this case, as described above.
40  * <p>
41  * If the application fails to restart the process when this exception is
42  * thrown, it is likely that re-opening the Environment will not be possible,
43  * or will result in unpredictable behavior. This is because the thread that
44  * stopped may be holding a resource that is needed by the newly opened
45  * Environment.
46  *
47  * @since 7.1
48  */
49 public class EnvironmentWedgedException extends EnvironmentFailureException {
