24 /**
25  * Starts a transaction, calls {@link TransactionWorker#doWork}, and handles
26  * transaction retry and exceptions.  To perform a transaction, the user
27  * implements the {@link TransactionWorker} interface and passes an instance of
28  * that class to the {@link #run run} method.
29  *
30  * <p>A single TransactionRunner instance may be used by any number of threads
31  * for any number of transactions.</p>
32  *
33  * <p>The behavior of the run() method depends on whether the environment is
34  * transactional, whether nested transactions are enabled, and whether a
35  * transaction is already active.</p>
36  *
37  * <ul>
38  * <li>When the run() method is called in a transactional environment and no
39  * transaction is active for the current thread, a new transaction is started
40  * before calling doWork().  If LockConflictException is thrown by doWork(),
41  * the transaction will be aborted and the process will be repeated up to the
42  * maximum number of retries.  If another exception is thrown by doWork() or
43  * the maximum number of retries has occurred, the transaction will be aborted
44  * and the exception will be rethrown by the run() method.  If no exception is
45  * thrown by doWork(), the transaction will be committed.  The run() method
46  * will not attempt to commit or abort a transaction if it has already been
47  * committed or aborted by doWork().</li>
48  *
49  * <li>When the run() method is called and a transaction is active for the
50  * current thread, and nested transactions are enabled, a nested transaction is
51  * started before calling doWork().  The transaction that is active when
52  * calling the run() method will become the parent of the nested transaction.
53  * The nested transaction will be committed or aborted by the run() method
54  * following the same rules described above.  Note that nested transactions may
55  * not be enabled for the JE product, since JE does not support nested
56  * transactions.</li>
57  *
58  * <li>When the run() method is called in a non-transactional environment, the
59  * doWork() method is called without starting a transaction.  The run() method
60  * will return without committing or aborting a transaction, and any exceptions
61  * thrown by the doWork() method will be thrown by the run() method.</li>
62  *
63  * <li>When the run() method is called and a transaction is active for the
64  * current thread and nested transactions are not enabled (the default) the
65  * same rules as above apply. All the operations performed by the doWork()
66  * method will be part of the currently active transaction.</li>
67  * </ul>
68  *
69  * <p>In a transactional environment, the rules described above support nested
70  * calls to the run() method and guarantee that the outermost call will cause
71  * the transaction to be committed or aborted.  This is true whether or not
72  * nested transactions are supported or enabled.  Note that nested transactions
73  * are provided as an optimization for improving concurrency but do not change
74  * the meaning of the outermost transaction.  Nested transactions are not
75  * currently supported by the JE product.</p>
76  *
77  * @author Mark Hayes
78  */

213     /**
214      * Calls the {@link TransactionWorker#doWork} method and, for transactional
215      * environments, may begin and end a transaction.  If the environment given
216      * is non-transactional, a transaction will not be used but the doWork()
217      * method will still be called.  See the class description for more
218      * information.
219      *
220      * @param worker the TransactionWorker.
221      *
222      * @throws LockConflictException when it is thrown by doWork() and the
223      * maximum number of retries has occurred.  The transaction will have been
224      * aborted by this method.
225      *
226      * @throws Exception when any other exception is thrown by doWork().  The
227      * exception will first be unwrapped by calling {@link
228      * ExceptionUnwrapper#unwrap}.  The transaction will have been aborted by
229      * this method.
230      */
231     public void run(TransactionWorker worker)
232         throws DatabaseException, Exception {
233 
234         if (currentTxn != null &&
235             (allowNestedTxn || currentTxn.getTransaction() == null)) {
236             /* Transactional and (not nested or nested txns allowed). */
237             int useMaxRetries = maxRetries;
238             for (int retries = 0;; retries += 1) {
239                 Transaction txn = null;
240                 try {
241                     txn = currentTxn.beginTransaction(config);
242                     worker.doWork();
243                     if (txn != null && txn == currentTxn.getTransaction()) {
244                         currentTxn.commitTransaction();
245                     }
246                     return;
247                 } catch (Throwable e) {
248                     e = ExceptionUnwrapper.unwrapAny(e);
249                     if (txn != null && txn == currentTxn.getTransaction()) {
250                         try {
251                             currentTxn.abortTransaction();
252                         } catch (Throwable e2) {
253 
254                             /*
255                              * We print this stack trace so that the
256                              * information is not lost when we throw the
257                              * original exception.
258                              */
259                             if (DbCompat.
260                                 TRANSACTION_RUNNER_PRINT_STACK_TRACES) {
261                                 e2.printStackTrace();
262                             }
263                             /* Force the original exception to be thrown. */
264                             retries = useMaxRetries;
265                         }
266                     }
267                     /* An Error should not require special handling. */
268                     if (e instanceof Error) {
269                         throw (Error) e;
270                     }
271                     /* Allow a subclass to determine retry policy. */
272                     Exception ex = (Exception) e;
273                     useMaxRetries =
274                         handleException(ex, retries, useMaxRetries);
275                     if (retries >= useMaxRetries) {
276                         throw ex;
277                     }
278                 }
279             }
280         } else {
281             /* Non-transactional or (nested and no nested txns allowed). */
282             try {
283                 worker.doWork();
284             } catch (Exception e) {
285                 throw ExceptionUnwrapper.unwrap(e);
286             }
287         }
288     }
