35 /**
36  * LogEntryType is an enumeration of all log entry types.
37  *
38  * <p>Log entries are versioned. When changing the persistent form of a log
39  * entry in any way that is incompatible with prior releases, make sure the
40  * LogEntry instance is capable of reading in older versions from the log and
41  * be sure to increment LOG_VERSION.  The LogEntry.readEntry and
42  * Loggable.readFromLog methods should check the actual version of the entry.
43  * If it is less than LOG_VERSION, the old version should be converted to the
44  * current version.
45  *
46  * <p>Prior to LOG_VERSION 6, each log entry type had a separate version number
47  * that was incremented only when that log version changed.  From LOG_VERSION 6
48  * onward, all types use the same version, the LOG_VERSION constant.  For
49  * versions prior to 6, the readEntry and readFromLog methods will be checking
50  * the old per-type version.  There is no overlap between the old per-type
51  * versions and the LOG_VERSION values, because the per-type values are all
52  * below 6. [#15365]</p>
53 
54  * <p>The LogEntry instance must be sure that older versions are converted in
55  * memory into a correct instance of the newest version, so when that LogEntry
56  * object is written again as the result of migration, eviction, the resulting
57  * new log entry conforms to the requirements of the new version.  If context
58  * objects are required for data conversion, the conversion can be done in the
59  * Node.postFetchInit method.</p>
60  *
61  * <p>Starting with LOG_VERSION 9, log entries that can be included in the
62  * replication stream must be able to write themselves in the format for the
63  * immediately previous log version, to allow replication during an upgrade
64  * when the master has been upgraded and a replica has not.  Starting with
65  * LOG_VERSION 8, log entries that support replication must implement {@link
66  * ReplicableLogEntry}.  When changes are made to replicable log entries for
67  * LOG_VERSION 9 and later, those entries need to support writing in the
68  * previous version's format.</p>
69  */

1021         @Override
1022         public boolean compareTypeAndVersion(int versionA,
1023                                              int versionB,
1024                                              byte typeB) {
1025             /* If the other entry is newer, the types should match. */
1026             if (versionA <= versionB && getTypeNum() == typeB) {
1027                 return true;
1028             }
1029 
1030             /*
1031              * If the other entry is older, the type might be an old,
1032              * deprecated, equivalent type.
1033              */
1034             if ((versionA > versionB) && isEquivalentOldType(typeB)) {
1035                 return true;
1036             }
1037 
1038             /*
1039              * In this case, the other entry's version is older or newer but it
1040              * doesn't matter -- it's completely the wrong type.
1041              */
1042             return false;
1043         }

1046             if ((!isTransactional() && (typeB == LOG_OLD_LN.getTypeNum())) ||
1047                 (isTransactional() &&
1048                  (typeB == LOG_OLD_LN_TRANSACTIONAL.getTypeNum()))) {
1049                 return true;
1050             }

1052             return false;
