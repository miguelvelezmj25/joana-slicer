22 /**
23  * A sub-interface of {@link LogEntry} that must be implemented by all log
24  * entries that can be replicated.  Replicable log entries are all those
25  * entries for which the associated {@link LogEntryType}'s {@link
26  * LogEntryType#isReplicationPossible} method returns {@code true}.  These are
27  * the log entries that can be included in the replication stream distributed
28  * from feeders to replicas during replication.  See [#22336].
29  *
30  * <p>Starting with the release using log version 9, as specified by {@link
31  * LogEntryType#LOG_VERSION_REPLICATE_OLDER}, all replicable log entries
32  * need to support writing themselves in earlier log formats, to support
33  * replication during an upgrade when the master is replicated first.  Any
34  * loggable objects that they reference should also implement {@link
35  * com.sleepycat.je.log.VersionedWriteLoggable} for the same reason.
36  *
37  * <p>The {@link #getLastFormatChange} method identifies the log version for
38  * which the entry's log format has most recently changed.  This information is
39  * used to determine if the current log format is compatible with a
40  * non-upgraded replica.
41  *
42  * <p>The {@link #getSize(int, boolean)} method overloading is used when
43  * creating the buffer that will be used to transmit the log entry data in the
44  * earlier format.
45  *
46  * <p>The {@link #writeEntry(ByteBuffer, int, boolean)} method overloading is
47  * used to convert the in-memory format of the log entry into the log data in
48  * the earlier format.
49  *
50  * <p>To simplify the implementation of writing log entries in multiple log
51  * version formats, a log entry that needs to be written in a previous format
52  * will first be read into its in-memory format in the current version, and
53  * then written from there to the previous format.
54  */
55 public interface ReplicableLogEntry extends LogEntry {
