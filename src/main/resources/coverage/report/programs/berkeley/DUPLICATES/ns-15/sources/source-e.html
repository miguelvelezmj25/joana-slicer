42 /**
43  * Obtains log files for a Replica from other members of the replication
44  * group. A Replica may need to do so if it has been offline for some time, and
45  * has fallen behind in its execution of the replication stream.
46  * <p>
47  * During that time, the connected nodes may have reduced their log files by
48  * deleting files after doing log cleaning. When this node rejoins the group,
49  * it is possible that the current Master's log files do not go back far enough
50  * to adequately {@link <a
51  * href="{@docRoot}/../ReplicationGuide/lifecycle.html#lifecycle-nodestartup">
52  * sync * up</a>} this node. In that case, the node can use a {@code
53  * NetworkRestore} object to copy the log files from one of the nodes in the
54  * group. The system tries to avoid deleting log files that either would be
55  * needed for replication by current nodes or where replication would be more
56  * efficient than network restore.
57  * <p>
58  * A Replica discovers the need for a NetworkRestore operation when a call to
59  * {@code ReplicatedEnvironment()} fails with a {@link
60  * InsufficientLogException}.
61  * <p>
62  * A call to {@code NetworkRestore.execute()} will copy the required log
63  * files from a member of the group who owns the files and seems to be the
64  * least busy. For example:
65  * <pre class=code>
66  *  try {
67  *     node = new ReplicatedEnvironment(envDir, envConfig, repConfig);
68  * } catch (InsufficientLogException insufficientLogEx) {
69  *
70  *     NetworkRestore restore = new NetworkRestore();
71  *     NetworkRestoreConfig config = new NetworkRestoreConfig();
72  *     config.setRetainLogFiles(false); // delete obsolete log files.
73  *
74  *     // Use the members returned by insufficientLogEx.getLogProviders() to
75  *     // select the desired subset of members and pass the resulting list
76  *     // as the argument to config.setLogProviders(), if the default selection
77  *     // of providers is not suitable.
78  *
79  *     restore.execute(insufficientLogEx, config);
80  *
81  *     // retry
82  *     node = new ReplicatedEnvironment(envDir, envConfig, repConfig);
83  * }
84  * </pre>
85  * @see <a href="{@docRoot}/../ReplicationGuide/logfile-restore.html">
86  * Restoring Log Files</a>
87  */
88 public class NetworkRestore {

121     /**
122      * Initializes this instance for an impending execute() operation.
123      *
124      * Algorithm
125      * =========
126      * If we simply choose the server with the highest maxVSLN, we would
127      * always choose the master, which is typically the server with the
128      * highest load. If we choose based on load alone, we may choose a
129      * lagging replica, and this may result in syncup failing later on the
130      * restored server. The compromise solution involves maxLag. We don't
131      * select servers less than maxLag behind the master (the server with the
132      * highest VLSN range end) to increase the chances of syncup working
133      * later, and among the non-lagging servers we choose the lowest load.
134      *
135      * 1. Collect list of servers and get their load/rangeEnd using the
136      *    first part of the restore protocol. For each server, its load is its
137      *    number of feeders and rangeEnd is the upper end of its VLSN range.
138      *    Remove unresponsive servers from the list.
139      *
140      * 2. At the beginning of each round, if the server list is empty, give up.
141      *
142      * 3. Sort list by load. Let minVLSN be max(all rangeEnds) minus maxLag.
143      *
144      * 4. Attempt to perform restore in list order, refreshing each server's
145      *    load/rangeEnd as we go. Reject any server with a refreshed rangeEnd
146      *    that is LT minVLSN or a refreshed load that is GT its prior known
147      *    load. Remove unresponsive servers from the list.
148      *
149      * 5. If the restore is incomplete, goto 2 and do another round.
150      *
151      * Note that between rounds the load and minVLSN of each server can
152      * change, which is why servers are not removed from the list unless
153      * they are unresponsive. The idea is to choose the best server based on
154      * the information we collected in the last round, but reject servers
155      * with new load or rangeEnd values that invalidate the earlier decision,
156      * and always get fresh load/rangeEnd values for each server.
157      *
158      * @param logException the exception packing information driving the
159      * restore operation.
160      * @param config may contain an explicit list of members.
161      * @return the list of candidate Server instances
162      * @throws IllegalArgumentException if the configured log providers are
163      * invalid
164      */
165     private List<Server> init(InsufficientLogException logException,
166                               NetworkRestoreConfig config)
167         throws IllegalArgumentException {
168 
169         repImpl = logException.getRepImpl();
170 
171         logger = LoggerUtils.getLogger(getClass());
172 
173         maxLag = repImpl.getConfigManager().getInt(
174             RepParams.NETWORKBACKUP_MAX_LAG);
175 
176         List<ReplicationNode> logProviders;
177 
178         if ((config.getLogProviders() != null) &&
179             (config.getLogProviders().size() > 0)) {
180             final Set<String> memberNames = new HashSet<>();
181             for (ReplicationNode node : logException.getLogProviders()) {
182                 memberNames.add(node.getName());
183             }
184             for (ReplicationNode node : config.getLogProviders()) {
185                 if (!memberNames.contains(node.getName())) {
186                     throw new  IllegalArgumentException
187                         ("Node:" + node.getName() +
188                          " is not a suitable member for NetworkRestore." +
189                          " It's not a member of logException." +
190                          "getLogProviders(): " +
191                          Arrays.toString(memberNames.toArray()));
192                 }
193             }
194             logProviders = config.getLogProviders();
195         } else {
196             logProviders = new LinkedList<>(logException.getLogProviders());
197         }
198 
199         LoggerUtils.info(logger, repImpl, "Started network restore");
200 
201         List<Server> serverList = new LinkedList<>();
202 
203         /*
204          * Set minVLSN and loadThreshold such that all attempts in the initial
205          * round will produce RejectedServerException. Real values will be used
206          * as servers are contacted and added to the list for the next round.
207          * No initial sort is needed because all servers have the same load.
208          */
209         VLSN maxVLSN = new VLSN(Long.MAX_VALUE);
210         int loadThreshold = -1;
211 
212         for (ReplicationNode node : logProviders) {
213             serverList.add(new Server(node, maxVLSN, loadThreshold));
214         }
215 
216         minVLSN = maxVLSN;
217 
218         return serverList;
219     }

221     /**
222      * Sorts the refreshed server list by load and computes minVLSN.
223      */
224     private void resetServerList(List<Server> serverList) {
225 
226         if (serverList.isEmpty()) {
227             return;
228         }
229 
230         /* Natural comparator sorts by Server.load. */
231         Collections.sort(serverList);
232 
233         /* Get server with max VLSN range end. */
234         Server maxVlsnServer = Collections.max(serverList,
235             Comparator.comparingLong(s -> s.rangeEnd.getSequence()));
236 
237         /* Subtract lag and ensure that result is GTE 0. */
238         minVLSN = new VLSN(
239             Math.max(0, maxVlsnServer.rangeEnd.getSequence() - maxLag));
240     }

242     /**
243      * Restores the log files from one of the members of the replication group.
244      * <p>
245      * If <code>config.getLogProviders()</code> returns null, or an empty list,
246      * it uses the member that is least busy as the provider of the log files.
247      * Otherwise it selects a member from the list, choosing the first member
248      * that's available, to provide the log files. If the members in this list
249      * are not present in <code>logException.getLogProviders()</code>, it will
250      * result in an <code>IllegalArgumentException</code> being thrown.
251      * Exceptions handlers for <code>InsufficientLogException</code> will
252      * typically use {@link InsufficientLogException#getLogProviders()} as the
253      * starting point to compute an appropriate list, with which to set up
254      * the <code>config</code> argument.
255      * <p>
256      * Log files that are currently at the node will be retained if they are
257      * part of a consistent set of log files. Obsolete log files are either
258      * deleted, or are renamed based on the the configuration of
259      * <code>config.getRetainLogFiles()</code>.
260      *
261      * @param logException the exception thrown by {@code
262      * ReplicatedEnvironment()} that necessitated this log refresh operation
263      *
264      * @param config configures the execution of the network restore operation
265      *
266      * @throws EnvironmentFailureException if an unexpected, internal or
267      * environment-wide failure occurs.
268      *
269      * @throws IllegalArgumentException if the <code>config</code> is invalid
270      *
271      * @see NetworkRestoreConfig
272      */
273     public synchronized void execute(InsufficientLogException logException,
274                                      NetworkRestoreConfig config)
275         throws EnvironmentFailureException,
276                IllegalArgumentException {
277 
278         try {
279             List<Server> serverList = init(logException, config);
280             boolean firstRound = true;
281             /*
282              * Loop trying busier servers. It sorts the servers by the number
283              * of active feeders at each server and contacts each one in turn,
284              * trying increasingly busy servers until it finds a suitable one
285              * that will service its request for log files. The same server may
286              * be contacted multiple times, since it may become busier between
287              * the time it was first contacted and a subsequent attempt.
288              */
289             while (!serverList.isEmpty()) {
290                 final List<Server> newServerList = new LinkedList<>();
291                 File envHome = repImpl.getEnvironmentHome();
292 
293                 for (Server server : serverList) {
294                     InetSocketAddress serverSocket =
295                         server.node.getSocketAddress();
296                     if (serverSocket.equals(repImpl.getSocket())) {
297                         /* Cannot restore from yourself. */
298                         continue;
299                     }
300                     LoggerUtils.info(logger, repImpl,
301                                      "Network restore candidate server: " +
302                                      server.node);
303                     logProvider = server.node;
304                     final long startTime = System.currentTimeMillis();
305                     try {
306                         backup = new NetworkBackup
307                             (serverSocket,
308                                 config.getReceiveBufferSize(),
309                                 envHome,
310                                 repImpl.getNameIdPair(),
311                                 config.getRetainLogFiles(),
312                                 server.load,
313                                 minVLSN,
314                                 repImpl,
315                                 repImpl.getFileManager(),
316                                 repImpl.getLogManager(),
317                                 repImpl.getChannelFactory(),
318                                 logException.getProperties());
319 
320                         backup.setInterruptHook(interruptHook);
321                         backup.execute();
322                         LoggerUtils.info(logger, repImpl, String.format(
323                             "Network restore completed from: %s. " +
324                                 "Elapsed time: %,d s.",
325                             server.node,
326                             ((System.currentTimeMillis() - startTime) /
327                                 1000)));
328                         return;
329                     } catch (RestoreMarker.FileCreationException e) {
330                         throw
331                             EnvironmentFailureException.unexpectedException(e);
332                     } catch (DatabaseException e) {
333                         /* Likely A malfunctioning server. */
334                         LoggerUtils.warning(logger, repImpl,
335                                             "Backup failed from node: " +
336                                             server.node + "\n" +
337                                             e.getMessage());
338                     } catch (ConnectException e) {
339                         /* Move on if the network connection is troublesome. */
340                         LoggerUtils.info(logger, repImpl,
341                                          "Backup server node: " + server.node +
342                                          " is not available: " +
343                                          e.getMessage());
344 
345                     } catch (IOException | ServiceConnectFailedException e) {
346                         /* Move on if the network connection is troublesome. */
347                         LoggerUtils.warning(logger, repImpl,
348                                             "Backup failed from node: " +
349                                             server.node + "\n" +
350                                             e.getMessage());
351                     } catch (RejectedServerException e) {
352                         /*
353                          * This is for one of two reasons:
354                          * 1. This is the initial round and we expect this
355                          *    exception for every server. We should not log a
356                          *    message. Add server to the new list now that we
357                          *    have its true rangeEnd and load.
358                          * 2. The server got busier or is lagging since the
359                          *    prior round, based on its refreshed rangeEnd and
360                          *    load. Add server to the list in case it qualifies
361                          *    in subsequent rounds.
362                          */
363                         if (!firstRound) {
364                             LoggerUtils.info(logger, repImpl, e.getMessage());
365                         }
366 
367                         newServerList.add(
368                             new Server(server.node, e.getRangeLast(),
369                                 e.getActiveServers()));
370 
371                     } catch (IllegalArgumentException e) {
372                         throw EnvironmentFailureException.unexpectedException(e);
373                     }
374                 }
375                 serverList = newServerList; /* New list for the next round. */
376                 resetServerList(serverList);
377                 firstRound = false;
378             }
379             throw EnvironmentFailureException.unexpectedState
380                 ("Tried and failed with every node");
381         } finally {
382             logException.releaseRepImpl();
383         }
384     }
