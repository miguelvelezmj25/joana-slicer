31 /**
32  * The FileProtector is primarily responsible for protecting files from being
33  * deleted due to log cleaning, when they are needed for other purposes. As
34  * such it is a gatekeeper for reading files. In addition it maintains certain
35  * metadata:
36  *  - the size of each file, which is needed for calculating disk usage;
37  *  - the first and last VLSNs in each reserved file, which are needed for
38  *    maintaining the VLSNIndex;
39  *  - the total size of active and reserved files, for disk usage statistics.
40  *
41  * All files are in three categories:
42  * + Active: readable by all. The minimum set of files needed to function.
43  * + Reserved: should be read only by feeders and low level utilities.
44  *   They have been cleaned and will not become active again. They can be
45  *   "condemned" and deleted if they are not being read.
46  * + Condemned: not readable and effectively invisible. They will be deleted
47  *   ASAP. They will not become active or reserved again. A file is typically
48  *   in the Condemned category for a very short time, while being deleted.
49  *
50  * Reserved files can be temporarily protected, i.e., prevented from being
51  * condemned and deleted. Only reserved files can be condemned and deleted. All
52  * active files are implicitly protected, but are also protected explicitly by
53  * consumers because they may become reserved while being consumed.
54  *
55  * Consumers of the File Protection Service
56  * ----------------------------------------
57  *
58  * + DiskOrderedScanner (DiskOrderedCursor and DatabaseCount)
59  *   - Protects all currently active files. By currently active we mean active
60  *     at the time they are protected. If they become reserved during the
61  *     scan, they must continue to be protected.
62  *   - Also protects any new files written during the scan.
63  *
64  * + DbBackup:
65  *   - Protects-and-lists all currently active files, or a subset of the
66  *     currently active files in the case of an incremental backup.
67  *   - Provides API to remove files from protected set as they are copied, to
68  *     allow file deletion.
69  *
70  * + NetworkRestore:
71  *   - Protects-and-lists all currently active files using DbBackup.
72  *   - Also protects the two newest reserved files at the time that the active
73  *     files are protected.
74  *   - Removes files from protected set as they are copied, to allow file
75  *     deletion.
76  *
77  * + Syncup:
78  *   - Protects all files (active and reserved) in an open ended range starting
79  *     with the file of the VLSNIndex range start. Barren files (with no
80  *     replicable entries) are not protected.
81  *
82  * + Feeder:
83  *   - Protects all files (active and reserved) in an open ended range starting
84  *     with the file of the current VLSN. Barren files (with no replicable
85  *     entries) are not protected.
86  *   - Advances lower bound of protected range as VLSN advances, to allow file
87  *     deletion.
88  *
89  * + Cleaner:
90  *   - Transforms active files into reserved files after cleaning them.
91  *   - Condemns and deletes reserved files to honor disk limits. Truncates head
92  *     of VLSNIndex when necessary to stay with disk thresholds.
93  *
94  * Syncup, Feeders and the VLSNIndex
95  * ---------------------------------
96  * During syncup, a ProtectedFileRange is used to protect files in the entire
97  * range of the VLSNIndex. Syncup must also prevent head truncation of the
98  * VLSNIndex itself because the file readers (used by both master and replica)
99  * use the VLSNIndex to position in the file at various times.
100  *
101  * A feeder file reader also protect all files from the current VLSN onward
102  * using a ProtectedFileRange. We rely on syncup to initialize the feeder's
103  * ProtectedFileRange safely, while the syncup ProtectedFileRange is in effect.
104  * The feeder reader will advance the lower bound of its ProtectedFileRange as
105  * it reads forward to allow files to be deleted. It also uses the VLNSIndex to
106  * skip over gaps in the file, although it is unclear whether this is really
107  * necessary.
108  *
109  * Therefore the syncup and feeder ProtectedFileRanges are special in that
110  * they also prevent head truncation of the VLSNIndex.
111  *
112  * The cleaner truncates the head of the VLSNIndex to allow deletion of files
113  * when necessary to stay within disk usage limits. This truncation must not
114  * be allowed when a syncup is in progress, and must not be allowed to remove
115  * the portion of the VLSN range used by a feeder. This is enforced using a
116  * special ProtectedFileRange (vlsnIndexRange) that protects the entire
117  * VLSNIndex range. The vlsnIndexRange is advanced when necessary to delete
118  * files that it protects, but only if those files are not protected by syncup
119  * or feeders. See {@link #checkVLSNIndexTruncation}, {@link
120  * com.sleepycat.je.rep.vlsn.VLSNTracker#tryTruncateFromHead} and {@link
121  * com.sleepycat.je.rep.vlsn.VLSNTracker#protectRangeHead}.
122  *
123  * We takes pains to avoid synchronizing on FileProtector while truncating the
124  * VLSNIndex head, which is a relatively expensive operation. (The
125  * FileProtector is meant to be used by multiple threads without a lot of
126  * blocking and should perform only a fairly small amount of work while
127  * synchronized.) The following approach is used to truncate the VLSNIndex head
128  * safely:
129  *
130  * -- To prevent disk usage limit violations, Cleaner.manageDiskUsage first
131  *    tries to delete reserved files without truncating the VLSNIndex. If this
132  *    is not sufficient, it then tries to truncate the VLSNIndex head. If the
133  *    VLSNIndex head can be truncated, then it tries again to delete reserved
134  *    files, since more files should then be unprotected.
135  *
136  * -- VLSNTracker synchronization is used to protect the VLSNIndex range. The
137  *    vlsnIndexRange ProtectedFileRange is advanced only while synchronized on
138  *    the VLSNTracker.
139  *
140  * -- VLSNTracker.tryTruncateFromHead (which is synchronized) calls
141  *    FileProtector.checkVLSNIndexTruncation to determine where to truncate the
142  *    index. Reserved files can be removed from the VLSNIndex range only if
143  *    they are not protected by syncup and feeders.
144  *
145  * -- The VLSNTracker range is then truncated, and the vlsnIndexRange is
146  *    advanced to allow file deletion, all while synchronized on the tracker.
147  *
148  * -- When a syncup starts, it adds a ProtectedFileRange with the same
149  *    startFile as the vlsnIndexRange. This is done while synchronized on the
150  *    VLSNTracker and it prevents the vlsnIndexRange from advancing during the
151  *    syncup.
152  *
153  * -- When a syncup is successful, on the master the Feeder is initialized and
154  *    it adds a ProtectedFileRange to protect the range of the VLSNIndex that
155  *    it is reading. This is done BEFORE removing the ProtectedFileRange that
156  *    was added at the start of the syncup. This guarantees that the files and
157  *    VLSNIndex range used by the feeder will not be truncated/deleted.
158  *
159  * Note that the special vlsnIndexRange ProtectedFileRange is excluded from
160  * LogSizeStats to avoid confusion and because this ProtectedFileRange does not
161  * ultimately prevent VLSNIndex head truncation or file deletion.
162  *
163  * Barren Files
164  * ------------
165  * Files with no replicable entries do not need to be protected by syncup or
166  * feeeders. See {@link #protectFileRange(String, long, boolean)}. Barren files
167  * may be created when cleaning is occurring but app writes are not, for
168  * example, when recovering from a cache size configuration error. In this
169  * situation it is important to delete the barren files to reclaim disk space.
170  *
171  * Such "barren" files are identified by having null begin/end VLSNs. The
172  * begin/end VLSNs for a file are part of the cleaner metadata that is
173  * collected when the cleaner processes a file. These VLSNs are for replicable
174  * entries only, not migrated entries that happen to contain a VLSN.
175  */

351

353                 final long file = files[i];
354 
355                 final File fileObj =
356                     new File(fileManager.getFullFileName(file));
357

359             }

361

364          * synchronization is required to get the last file number. Do not

472      * reserved file and returns it. If the returned file cannot be deleted by
473      * the caller, {@link #putBackCondemnedFile} should be called so the file
474      * deletion can be retried later.

478      *
479      * @return {file, size} pair or null if a condemned file is not available.

481     synchronized Pair<Long, Long> takeCondemnedFile(long fromFile) {
482 
483         if (!condemnedFiles.isEmpty()) {
484             final Long file = condemnedFiles.firstKey();
485             final Long size = condemnedFiles.remove(file);
486             return new Pair<>(file, size);
487         }
488 
489         if (reservedFiles.isEmpty()) {
490             return null;
491         }
492 
493         fileLoop:
494         for (final Map.Entry<Long, ReservedFileInfo> entry :
495                 reservedFiles.tailMap(fromFile).entrySet()) {
496 
497             final Long file = entry.getKey();
498             final ReservedFileInfo info = entry.getValue();
499 
500             for (final ProtectedFileSet pfs : protectedFileSets.values()) {
501                 if (pfs.isProtected(file, info)) {
502                     continue fileLoop;
503                 }
504             }
505 
506             reservedFiles.remove(file);
507             return new Pair<>(file, info.size);
508         }
509 
510         return null;
511     }

542     synchronized LogSizeStats getLogSizeStats() {
543 
544         /* Calculate active size. */
545         final NavigableMap<Long, Long> activeFiles = getActiveFiles();
546         long activeSize = 0;
547 
548         for (final long size : activeFiles.values()) {
549             activeSize += size;
550         }
551 
552         /* Add size of last file, which is not included in activeFiles. */
553         final long lastFileNum = activeFiles.isEmpty() ?
554             0 : activeFiles.lastKey() + 1;
555 
556         final File lastFile = new File(
557             envImpl.getFileManager().getFullFileName(lastFileNum));
558 
559         if (lastFile.exists()) {
560             activeSize += lastFile.length();
561         }
562 
563         /* Calculate reserved and protected sizes. */
564         long reservedSize = 0;
565         long protectedSize = 0;
566         final Map<String, Long> protectedSizeMap = new HashMap<>();
567 
568         for (final Map.Entry<Long, ReservedFileInfo> entry :
569                 reservedFiles.entrySet()) {
570 
571             final Long file = entry.getKey();
572             final ReservedFileInfo info = entry.getValue();
573             reservedSize += info.size;
574             boolean isProtected = false;
575 
576             for (final ProtectedFileSet pfs : protectedFileSets.values()) {
577 
578                 if (pfs == vlsnIndexRange || !pfs.isProtected(file, info)) {
579                     continue;
580                 }
581 
582                 isProtected = true;
583 
584                 protectedSizeMap.compute(
585                     pfs.getName(),
586                     (k, v) -> ((v != null) ? v : 0) + info.size);
587             }
588 
589             if (isProtected) {
590                 protectedSize += info.size;
591             }
592         }
593 
594         return new LogSizeStats(
595             activeSize, reservedSize, protectedSize, protectedSizeMap);
596     }

622      * advance. The deleteEnd returned will be less than preserveVLSN.
623      *
624      * @return {deleteEnd, deleteFileNum} pair if the protected file range
625      * should be advanced, or null if advancing is not currently possible.
626      *  -- deleteEnd is the last VLSN to be truncated.
627      *  -- deleteFileNum the file having deleteEnd as its last VLSN.
628      */
629     public synchronized Pair<VLSN, Long> checkVLSNIndexTruncation(
630         final long bytesNeeded,
631         final VLSN preserveVLSN) {
632 
633         /*
634          * Determine how many reserved files we need to delete, and find the
635          * last file/VLSN in that set of files, which is the truncation point.
636          */
637         VLSN truncateVLSN = VLSN.NULL_VLSN;
638         long truncateFile = -1;
639         long deleteBytes = 0;
640 
641         fileLoop:
642         for (final Map.Entry<Long, ReservedFileInfo> entry :
643                 reservedFiles.entrySet()) {
644 
645             final Long file = entry.getKey();
646             final ReservedFileInfo info = entry.getValue();
647 
648             for (final ProtectedFileSet pfs : protectedFileSets.values()) {
649 
650                 if (pfs == vlsnIndexRange || !pfs.protectVlsnIndex) {
651                     continue;
652                 }
653 
654                 if (pfs.isProtected(file, info)) {
655                     break fileLoop;
656                 }
657             }
658 
659             final VLSN lastVlsn = info.endVLSN;
660 
661             if (!lastVlsn.isNull()) {
662                 if (lastVlsn.compareTo(preserveVLSN) > 0) {
663                     break;
664                 }
665                 truncateVLSN = lastVlsn;
666                 truncateFile = file;
667             }
668 
669             deleteBytes += info.size;
670 
671             if (deleteBytes >= bytesNeeded) {
672                 break;
673             }
674         }
675 
676         return truncateVLSN.isNull() ? null :
677             new Pair<>(truncateVLSN, truncateFile);
678     }

681      * A ProtectedFileSet is used to prevent a set of files from being deleted.
682      * Implementations must meet two special requirements:
683      *
684      * 1. After a ProtectedFileSet is added using {@link #addFileProtection},
685      * its set of protected files (the set for which {@link #isProtected}
686      * returns true) may only be changed by shrinking the set. Files may not be
687      * added to the set of protected files. (One exception is that newly
688      * created files are effectively to a file set defined as an opened ended
689      * range.)
690      *
691      * 2. Shrinking the protected set can be done without synchronization on
692      * FileProtector. However, implementations should ensure that changes made
693      * in one thread are visible to all threads.
694      *
695      * The intention is to allow protecting a set of files that are to be
696      * processed in some way, and allow easily shrinking this set as the files
697      * are processed, so that the processed files may be deleted. Changes to
698      * the protected set should be visible to all threads so that periodic disk
699      * space reclamation tasks can delete unprotected files ASAP. {@link
700      * ProtectedFileRange} is a simple class that meets these requirements.
701      */

715          * across all file sets added to the FileProtector.
716          */

719         }
720 
721         /**

723          */
