739     /**
740      * When multiple operations are performed, CacheMode-based eviction is
741      * performed for a given operation at the end of the next operation, which
742      * calls close() or reset() on the CursorImpl of the previous operation.
743      * Eviction for the last operation (including when only one operation is
744      * performed) also occurs during Cursor.close(), which calls
745      * CursorImpl.close().
746      *
747      * By default, the CacheMode returned by DatabaseImpl.getCacheMode is used,
748      * and the defaults specified by the user for the Database or Environment
749      * are applied.  However, the default mode can be overridden by the user by
750      * calling Cursor.setCacheMode, and the mode may be changed prior to each
751      * operation, if desired.
752      *
753      * To implement a per-operation CacheMode, two CacheMode fields are
754      * maintained.  Cursor.cacheMode is the mode to use for the next operation.
755      * CursorImpl.cacheMode is the mode that was used for the previous
756      * operation, and that is used for eviction when that CursorImpl is closed
757      * or reset.
758      *
759      * This method must be called with the BIN latched but may release it,
760      * namely when the BIN is evicted.
761      */
762     private void performCacheModeEviction(final CursorImpl newCursor) {
763 
764         /* Short circuit modes that do not perform LN or BIN eviction. */
765         if (cacheMode == CacheMode.DEFAULT ||
766             cacheMode == CacheMode.KEEP_HOT) {
767             return;
768         }
769 
770         final boolean movedOffBin;
771         final boolean movedOffLn;
772 
773         if (newCursor != null) {
774             movedOffBin = (bin != newCursor.bin);
775             movedOffLn = (movedOffBin || index != newCursor.index);
776         } else {
777             movedOffBin = true;
778             movedOffLn = true;
779         }
780 
781         if (movedOffLn) {
782             performCacheModeLNEviction();
783         }
784 
785         /* Short circuit modes that do not perform BIN eviction. */
786         if (cacheMode == CacheMode.EVICT_LN) {
787             return;
788         }
789 
790         if (movedOffBin) {
791             performCacheModeBINEviction(bin);
792         }
793     }

998                 }
999             }

1003             setStorageSize();

1006             success = true;
1007

1009 
1010             return DbInternal.makeResult(expiration, expirationInHours);
1011 
1012         } finally {
1013 
1014             if (success &&
1015                 !dbImpl.isInternalDb() &&
1016                 bin != null &&
1017                 bin.isBINDelta()) {
1018                 dbImpl.getEnv().incBinDeltaDeletes();
1019             }
1020 
1021             releaseBIN();
1022         }

3404      * LN, unless locking is not desired.

3407      * Locker.nonBlockingLock rather than this method), as long as the lock is
3408      * released before the BIN latch is released.  In this case a null locker
3409      * is passed to the LN logging method; locking the new LSN is unnecessary
3410      * because no other thread can access the new LSN until the BIN latch is
3411      * released.

3414      * used by the cleaner, when flushing dirty deferred-write LNs, and by

3418      * -----------------------------
3419      * The allowUncontended param is passed as true for update and delete
3420      * operations as an optimization for the case where no lock on the old LSN
3421      * is held by any locker.  In this case we don't need to lock the old LSN
3422      * at all, as long as we log the new LSN before releasing the BIN latch.
3423      *
3424      * 1. Latch BIN
3425      * 2. Determine that no lock/waiter exists for oldLsn
3426      * 3. Log LN and get lsn
3427      * 4. Lock lsn
3428      * 5. Update BIN

3432      * assumption is that another locker will first have to latch the BIN to
3433      * get oldLsn, before requesting a lock.

3436      * before requesting the lock.

3441      * Step 1 and 2

3449      * using a lockLN method in this class, which will release the lock and
3450      * retry if the LSN changes while acquiring the lock.  Because it
3451      * re-latches the BIN to check the LSN, this will serialize access to the
3452      * LSN for locking, guaranteeing that two conflicting locks cannot be
3453      * granted on the old and new LSNs.
