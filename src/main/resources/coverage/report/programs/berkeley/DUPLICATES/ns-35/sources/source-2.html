31 /**
32  * Provides access to the raw data in a store for use by general purpose tools.
33  * A <code>RawStore</code> provides access to stored entities without using
34  * entity classes or key classes.  Keys are represented as simple type objects
35  * or, for composite keys, as {@link RawObject} instances, and entities are
36  * represented as {@link RawObject} instances.
37  *
38  * <p>{@code RawStore} objects are thread-safe.  Multiple threads may safely
39  * call the methods of a shared {@code RawStore} object.</p>
40  *
41  * <p>When using a {@code RawStore}, the current persistent class definitions
42  * are not used.  Instead, the previously stored metadata and class definitions
43  * are used.  This has several implications:</p>
44  * <ol>
45  * <li>An {@code EntityModel} may not be specified using {@link
46  * StoreConfig#setModel}.  In other words, the configured model must be
47  * null (the default).</li>
48  * <li>When storing entities, their format will not automatically be evolved
49  * to the current class definition, even if the current class definition has
50  * changed.</li>
51  * </ol>
52  *
53  * @author Mark Hayes
54  */
