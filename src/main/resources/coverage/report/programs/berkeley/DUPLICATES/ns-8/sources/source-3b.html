18 /**
19  * The operation type passed to "put" methods on databases and cursors.
20  */
21 public enum Put {
22 
23     /**
24      * Inserts or updates a record depending on whether a matching record is
25      * already present.
26      *
27      * <p>If the database does not have duplicate keys, a matching record is
28      * defined as one with the same key. The existing record's data will be
29      * replaced. In addition, if a custom key comparator is configured, and the
30      * key bytes are different but considered equal by the comparator, the key
31      * is replaced.</p>
32      *
33      * <p>If the database does have duplicate keys, a matching record is
34      * defined as one with the same key and data. As above, if a custom key
35      * comparator is configured, and the key bytes are different but considered
36      * equal by the comparator, the key is replaced. In addition, if a custom
37      * duplicate comparator is configured, and the data bytes are different but
38      * considered equal by the comparator, the data is replaced.</p>
39      *
40      * <p>The operation always succeeds (null is never returned).</p>
41      */
42     OVERWRITE(PutMode.OVERWRITE),
43 
44     /**
45      * Inserts a record if a record with a matching key is not already present.
46      *
47      * <p>If the database has duplicate keys, a record is inserted only if
48      * there are no records with a matching key.</p>
49      *
50      * <p>The operation does not succeed (null is returned) when an existing
51      * record matches.</p>
52      */
53     NO_OVERWRITE(PutMode.NO_OVERWRITE),
54 
55     /**
56      * Inserts a record in a database with duplicate keys if a record with a
57      * matching key and data is not already present.
58      *
59      * <p>This operation is not allowed for databases that do not have
60      * duplicate keys.</p>
61      *
62      * <p>The operation does not succeed (null is returned) when an existing
63      * record matches.</p>
64      */
65     NO_DUP_DATA(PutMode.NO_DUP_DATA),
66 
67     /**
68      * Updates the data of the record at the cursor position.
69      *
70      * <p>If the database does not have duplicate keys, the existing record's
71      * data will be replaced.</p>
72      *
73      * <p>If the database does have duplicate keys, the existing data is
74      * replaced but it is must be considered equal by the duplicate comparator.
75      * If the data is not considered equal, {@link DuplicateDataException} is
76      * thrown. Using the default comparator, a key is considered equal only if
77      * its bytes are equal. Therefore, changing the data is only possible if a
78      * custom duplicate comparator is configured.</p>
79      *
80      * <p>A <a href="Cursor.html#partialEntry">partial data item</a> may be
81      * specified to optimize for partial data update.</p>
82      *
83      * <p>This operation cannot be used to update the key of an existing record
84      * and in fact the key parameter must be null when calling generic put
85      * methods such as
86      * {@link Database#put(Transaction, DatabaseEntry, DatabaseEntry, Put,
87      * WriteOptions)} and
88      * {@link Cursor#put(DatabaseEntry, DatabaseEntry, Put, WriteOptions)}.</p>
89      *
90      * <p>The operation does not succeed (null is returned) if the record at
91      * the current position has been deleted. This can occur in two cases: 1.
92      * If the record was deleted using this cursor and then accessed. 2. If the
93      * record was not locked by this cursor or transaction, and was deleted by
94      * another thread or transaction after this cursor was positioned on
95      * it.</p>
96      */
97     CURRENT(PutMode.CURRENT);
