19 /**
20  * Provides exclusive (mutex-like) latching. This is implemented with Java's
21  * ReentrantLock, which is extended for a few reasons:
22  * 1) To prevent reentrant use, since latches are not used reentrantly in JE.
23  * This increases reliability by detecting accidental reentrant calls.
24  * Checks for reentrancy are unconditional, i.e., checked in production.
25  * 2) To support instrumentation for debugging (see LatchSupport).
26  * 3) Automatic use of configured latch timeout.
27  * 4) Built-in thread interrupt handling.
28  *
29  * Latches are expected to be held for short, defined periods of time.  No
30  * deadlock detection is provided so it is the caller's responsibility to
31  * sequence latch acquisition in an ordered fashion to avoid deadlocks.
32  */
33 public interface Latch {

38      * @throws EnvironmentFailureException if the latch is already held by the
39      * calling thread.
40      */

45      * not available.
