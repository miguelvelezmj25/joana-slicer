35 /**
36  * The base class for all object formats.  Formats are used to define the
37  * stored layout for all persistent classes, including simple types.
38  *
39  * The design documentation below describes the storage format for entities and
40  * its relationship to information stored per format in the catalog.
41  *
42  * Requirements
43  * ------------
44  * + Provides EntityBinding for objects and EntryBinding for keys.
45  * + Provides SecondaryKeyCreator, SecondaryMultiKeyCreator and
46  *   SecondaryMultiKeyNullifier (SecondaryKeyNullifier is redundant).
47  * + Works with reflection and bytecode enhancement.
48  * + For reflection only, works with any entity model not just annotations.
49  * + Bindings are usable independently of the persist API.
50  * + Performance is almost equivalent to hand coded tuple bindings.
51  * + Small performance penalty for compatible class changes (new fields,
52  *   widening).
53  * + Secondary key create/nullify do not have to deserialize the entire record;
54  *   in other words, store secondary keys at the start of the data.
55  *
56  * Class Format
57  * ------------
58  * Every distinct class format is given a unique format ID.  Class IDs are not
59  * equivalent to class version numbers (as in the version property of @Entity
60  * and @Persistent) because the format can change when the version number does
61  * not.  Changes that cause a unique format ID to be assigned are:
62  *
63  * + Add field.
64  * + Widen field type.
65  * + Change primitive type to primitive wrapper class.
66  * + Add or drop secondary key.
67  * + Any incompatible class change.
68  *
69  * The last item, incompatible class changes, also correspond to a class
70  * version change.
71  *
72  * For each distinct class format the following information is conceptually
73  * stored in the catalog, keyed by format ID.
74  *
75  * - Class name
76  * - Class version number
77  * - Superclass format
78  * - Kind: simple, enum, complex, array
79  * - For kind == simple:
80  *     - Primitive class
81  * - For kind == enum:
82  *     - Array of constant names, sorted by name.
83  * - For kind == complex:
84  *     - Primary key fieldInfo, or null if no primary key is declared
85  *     - Array of secondary key fieldInfo, sorted by field name
86  *     - Array of other fieldInfo, sorted by field name
87  * - For kind == array:
88  *     - Component class format
89  *     - Number of array dimensions
90  * - Other metadata for RawType
91  *
92  * Where fieldInfo is:
93  *     - Field name
94  *     - Field class
95  *     - Other metadata for RawField
96  *
97  * Data Layout
98  * -----------
99  * For each entity instance the data layout is as follows:
100  *
101  *   instanceData: formatId keyFields... nonKeyFields...
102  *   keyFields:    fieldValue...
103  *   nonKeyFields: fieldValue...
104  *
105  * The formatId is the (positive non-zero) ID of a class format, defined above.
106  * This is ID of the most derived class of the instance.  It is stored as a
107  * packed integer.
108  *
109  * Following the format ID, zero or more sets of secondary key field values
110  * appear, followed by zero or more sets of other class field values.
111  *
112  * The keyFields are the sets of secondary key fields for each class in order
113  * of the highest superclass first.  Within a class, fields are ordered by
114  * field name.
115  *
116  * The nonKeyFields are the sets of other non-key fields for each class in
117  * order of the highest superclass first.  Within a class, fields are ordered
118  * by field name.
119  *
120  * A field value is:
121  *
122  *   fieldValue:   primitiveValue
123  *               | nullId
124  *               | instanceRef
125  *               | instanceData
126  *               | simpleValue
127  *               | enumValue
128  *               | arrayValue
129  *
130  * For a primitive type, a primitive value is used as defined for tuple
131  * bindings.  For float and double, sorted float and sorted double tuple values
132  * are used.
133  *
134  * For a non-primitive type with a null value, a nullId is used that has a zero
135  * (illegal formatId) value.  This includes String and other simple reference
136  * types.  The formatId is stored as a packed integer, meaning that it is
137  * stored as a single zero byte.
138  *
139  * For a non-primitive type, an instanceRef is used for a non-null instance
140  * that appears earlier in the data byte array.  An instanceRef is the negation
141  * of the byte offset of the instanceData that appears earlier.  It is stored
142  * as a packed integer.
143  *
144  * The remaining rules apply only to reference types with non-null values that
145  * do not appear earlier in the data array.
146  *
147  * For an array type, an array formatId is used that identifies the component
148  * type and the number of array dimensions.  This is followed by an array
149  * length (stored as a packed integer) and zero or more fieldValue elements.
150  * For an array with N+1 dimensions where N is greater than zero, the leftmost
151  * dimension is enumerated such that each fieldValue element is itself an array
152  * of N dimensions or null.
153  *
154  *   arrayValue:  formatId length fieldValue...
155  *
156  * For an enum type, an enumValue is used, consisting of a formatId that
157  * identifies the enum class and an enumIndex (stored as a packed integer) that
158  * identifies the constant name in the enum constant array of the enum class
159  * format:
160  *
161  *   enumValue:   formatId enumIndex
162  *
163  * For a simple type, a simpleValue is used.  This consists of the formatId
164  * that identifies the class followed by the simple type value.  For a
165  * primitive wrapper type the simple type value is the corresponding primitive,
166  * for a Date it is the milliseconds as a long primitive, and for BigInteger or
167  * BigDecimal it is a byte array as defined for tuple bindings of these types.
168  *
169  *   simpleValue: formatId value
170  *
171  * For all other complex types, an instanceData is used, which is defined
172  * above.
173  *
174  * Secondary Keys
175  * --------------
176  * For secondary key support we must account for writing and nullifying
177  * specific keys.  Rather than instantiating the entity and then performing
178  * the secondary key operation, we strive to perform the secondary key
179  * operation directly on the byte format.
180  *
181  * To create a secondary key we skip over other fields and then copy the bytes
182  * of the embedded key.  This approach is very efficient because a) the entity
183  * is not instantiated, and b) the secondary keys are stored at the beginning
184  * of the byte format and can be quickly read.
185  *
186  * To nullify we currently instantiate the raw entity, set the key field to null
187  * (or remove it from the array/collection), and convert the raw entity back to
188  * bytes.  Although the performance of this approach is not ideal because it
189  * requires serialization, it avoids the complexity of modifying the packed
190  * serialized format directly, adjusting references to key objects, etc.  Plus,
191  * when we nullify a key we are going to write the record, so the serialization
192  * overhead may not be significant.  For the record, I tried implementing
193  * nullification of the bytes directly and found it was much too complex.
194  *
195  * Lifecycle
196  * ---------
197  * Format are managed by a Catalog class.  Simple formats are managed by
198  * SimpleCatalog, and are copied from the SimpleCatalog by PersistCatalog.
199  * Other formats are managed by PersistCatalog.  The lifecycle of a format
200  * instance is:
201  *
202  * - Constructed by the catalog when a format is requested for a Class
203  *   that currently has no associated format.
204  *
205  * - The catalog calls setId() and adds the format to its format list
206  *   (indexed by format id) and map (keyed by class name).
207  *
208  * - The catalog calls collectRelatedFormats(), where a format can create
209  *   additional formats that it needs, or that should also be persistent.
210  *
211  * - The catalog calls initializeIfNeeded(), which calls the initialize()
212  *   method of the format class.
213  *
214  * - initialize() should initialize any transient fields in the format.
215  *   initialize() can assume that all related formats are available in the
216  *   catalog.  It may call initializeIfNeeded() for those related formats, if
217  *   it needs to interact with an initialized related format; this does not
218  *   cause a cycle, because initializeIfNeeded() does nothing for an already
219  *   initialized format.
220  *
221  * - The catalog creates a group of related formats at one time, and then
222  *   writes its entire list of formats to the catalog DB as a single record.
223  *   This grouping reduces the number of writes.
224  *
225  * - When a catalog is opened and the list of existing formats is read.  After
226  *   a format is deserialized, its initializeIfNeeded() method is called.
227  *   setId() and collectRelatedFormats() are not called, since the ID and
228  *   related formats are stored in serialized fields.
229  *
230  * - There are two modes for opening an existing catalog: raw mode and normal
231  *   mode.  In raw mode, the old format is used regardless of whether it
232  *   matches the current class definition; in fact the class is not accessed
233  *   and does not need to be present.
234  *
235  * - In normal mode, for each existing format that is initialized, a new format
236  *   is also created based on the current class and metadata definition.  If
237  *   the two formats are equal, the new format is discarded.  If they are
238  *   unequal, the new format becomes the current format and the old format's
239  *   evolve() method is called.  evolve() is responsible for adjusting the
240  *   old format for class evolution.  Any number of non-current formats may
241  *   exist for a given class, and are setup to evolve the single current format
242  *   for the class.
243  *
244  * @author Mark Hayes
245  */

598         Class stype = type.getSuperclass();
599         while (stype != null && stype != Object.class) {

601             addInterfaces(stype);
602             stype = stype.getSuperclass();
603         }
604     }

612             if (iface != Enhanced.class) {
613                 supertypes.add(iface.getName());

1000     public String toString() {
1001         final String INDENT = "  ";
1002         final String INDENT2 = INDENT + "  ";
1003         StringBuilder buf = new StringBuilder(500);
1004         if (isSimple()) {
1005             addTypeHeader(buf, "SimpleType");
1006             buf.append(" primitive=\"");

1008             buf.append("\"/>\n");

1010             addTypeHeader(buf, "EnumType");
1011             buf.append(">\n");

1013                 buf.append(INDENT);
1014                 buf.append("<Constant>");
1015                 buf.append(constant);
1016                 buf.append("</Constant>\n");
1017             }
1018             buf.append("</EnumType>\n");
1019         } else if (isArray()) {
1020             addTypeHeader(buf, "ArrayType");
1021             buf.append(" componentId=\"");
1022             buf.append(getComponentType().getVersion());
1023             buf.append("\" componentClass=\"");
1024             buf.append(getComponentType().getClassName());
1025             buf.append("\" dimensions=\"");
1026             buf.append(getDimensions());
1027             buf.append("\"/>\n");
1028         } else {
1029             addTypeHeader(buf, "ComplexType");
1030             Format superType = getSuperType();
1031             if (superType != null) {
1032                 buf.append(" superTypeId=\"");
1033                 buf.append(superType.getId());
1034                 buf.append("\" superTypeClass=\"");
1035                 buf.append(superType.getClassName());
1036                 buf.append('"');
1037             }
1038             Format proxiedFormat = getProxiedFormat();
1039             if (proxiedFormat != null) {
1040                 buf.append(" proxiedTypeId=\"");
1041                 buf.append(proxiedFormat.getId());
1042                 buf.append("\" proxiedTypeClass=\"");
1043                 buf.append(proxiedFormat.getClassName());
1044                 buf.append('"');
1045             }
1046             PrimaryKeyMetadata priMeta = null;
1047             Map<String, SecondaryKeyMetadata> secondaryKeys = null;
1048             List<FieldMetadata> compositeKeyFields = null;
1049             ClassMetadata clsMeta = getClassMetadata();
1050             if (clsMeta != null) {
1051                 compositeKeyFields = clsMeta.getCompositeKeyFields();
1052                 priMeta = clsMeta.getPrimaryKey();
1053                 secondaryKeys = clsMeta.getSecondaryKeys();
1054             }
1055             buf.append(" kind=\"");
1056             buf.append(isEntity() ? "entity" :
1057                        ((compositeKeyFields != null) ? "compositeKey" :
1058                         "persistent"));
1059             buf.append("\">\n");
1060             Map<String, RawField> fields = getFields();
1061             if (fields != null) {
1062                 for (RawField field : fields.values()) {
1063                     String name = field.getName();
1064                     RawType type = field.getType();
1065                     buf.append(INDENT);
1066                     buf.append("<Field");
1067                     buf.append(" name=\"");
1068                     buf.append(name);
1069                     buf.append("\" typeId=\"");
1070                     buf.append(type.getId());
1071                     buf.append("\" typeClass=\"");
1072                     buf.append(type.getClassName());
1073                     buf.append('"');
