18 /**
19  * <!-- begin JE only -->
20  * @hidden
21  * <!-- end JE only -->
22  * Determines the file names to use for primary and secondary databases.
23  *
24  * <p>Each {@link PrimaryIndex} and {@link SecondaryIndex} is represented
25  * internally as a Berkeley DB {@link Database}.  The file names of primary and
26  * secondary indices must be unique within the environment, so that each index
27  * is stored in a separate database file.</p>
28  *
29  * <p>By default, the file names of primary and secondary databases are
30  * defined as follows.</p>
31  *
32  * <p>The syntax of a primary index database file name is:</p>
33  * <pre>   STORE_NAME-ENTITY_CLASS</pre>
34  * <p>Where STORE_NAME is the name parameter passed to {@link
35  * EntityStore#EntityStore EntityStore} and ENTITY_CLASS is name of the class
36  * passed to {@link EntityStore#getPrimaryIndex getPrimaryIndex}.</p>
37  *
38  * <p>The syntax of a secondary index database file name is:</p>
39  * <pre>   STORE_NAME-ENTITY_CLASS-KEY_NAME</pre>
40  * <p>Where KEY_NAME is the secondary key name passed to {@link
41  * EntityStore#getSecondaryIndex getSecondaryIndex}.</p>
42  *
43  * <p>The default naming described above is implemented by the built-in {@link
44  * DatabaseNamer#DEFAULT} object.  An application may supply a custom {@link
45  * DatabaseNamer} to overrride the default naming scheme.  For example, a
46  * custom namer could place all database files in a subdirectory with the name
47  * of the store.  A custom namer could also be used to name files according to
48  * specific file system restrictions.</p>
49  *
50  * <p>The custom namer object must be an instance of the {@code DatabaseNamer}
51  * interface and is configured using {@link StoreConfig#setDatabaseNamer
52  * setDatabaseNamer}.</p>
53  *
54  * <p>When copying or removing all databases in a store, there is one further
55  * consideration.  There are two internal databases that must be kept with the
56  * other databases in the store in order for the store to be used.  These
57  * contain the data formats and sequences for the store.  Their entity class
58  * names are:</p>
59  *
60  * <pre>   com.sleepycat.persist.formats</pre>
61  * <pre>   com.sleepycat.persist.sequences</pre>
62  *
63  * <p>With default database naming, databases with the following names will be
64  * present each store.</p>
65  *
66  * <pre>   STORE_NAME-com.sleepycat.persist.formats</pre>
67  * <pre>   STORE_NAME-com.sleepycat.persist.sequences</pre>
68  *
69  * <p>These databases must normally be included with copies of other databases
70  * in the store.  They should not be modified by the application.</p>
71  */

79      *
80      * @param entityClassName the complete name of the entity class for a
81      * primary or secondary index.
82      *
83      * @param keyName the key name identifying a secondary index, or null for
84      * a primary index.
85      *
86      * @return the file name.
87      */
88     public String getFileName(String storeName,
89         String entityClassName,
90         String keyName);
91 
92     /**
93      * The default database namer.
94      *
95      * <p>The {@link #getFileName getFileName} method of this namer returns the
96      * {@code storeName}, {@code entityClassName} and {@code keyName}
97      * parameters as follows:</p>
98      *
99      * <pre class="code">
100      * if (keyName != null) {
101      *     return storeName + '-' + entityClassName + '-' + keyName;
102      * } else {
103      *     return storeName + '-' + entityClassName;
104      * }</pre>
105      */
106     public static final DatabaseNamer DEFAULT = new DatabaseNamer() {
107 
108         public String getFileName(String storeName,
109                                   String entityClassName,
110                                   String keyName) {
111             if (keyName != null) {
112                 return storeName + '-' + entityClassName + '-' + keyName;
113             } else {
114                 return storeName + '-' + entityClassName;
115             }
116         }
117     };
