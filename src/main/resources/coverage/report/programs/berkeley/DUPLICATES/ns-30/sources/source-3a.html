26 /**
27  * The Stats infrastructure provides context for JE statistics. Each statistic
28  * has these attributes:
29  * - metadata - specifically, a name and description
30  * - each statistic is associated with a parent stat group, which itself has
31  *   a name and description.
32  * - support for the StatsConfig.clear semantics
33  * - a way to print statistics in a user friendly way.
34  *
35  * To create a statistic variable, instantiate one of the concrete subclasses
36  * of Stat. Each concrete subclass should hold the methods that are needed to
37  * best set and display the value. For example, instead of using LongStat to
38  * hold a timestamp or LSN value, use TimestampStat or LSNStat. A Stat instance
39  * needs to specify a StatDefinition. There may be multiple Stat variables in
40  * different components that share a StatDefinition. They are differentiated
41  * when displayed by their parent StatGroup.
42  *
43  * Each Stat instance is associated with a StatGroup, which holds the
44  * collection of stats that belong to a given component. Each member of the
45  * StatGroup has a unique StatDefinition. StatGroups can be combined, in order
46  * to accumulate values. For example, the LockManager may have multiple lock
47  * tables. Each lock table keeps its own latch statistics. When LockStats are
48  * generated, the StatsGroup for each latch is collected and rolled up into a
49  * single StatGroup, using the addAll(StatGroup) method.
50  *
51  * The Stats infrastructure is for internal use only. Public API classes like
52  * EnvironmentStats, LockStats, etc, contain StatGroups. A call to retrieve
53  * stats is implemented by getting a clone of the StatGroups held by the
54  * components like the cleaner, the incompressor, the LockManager, etc.  The
55  * public API classes provide getter methods that reach into the StatGroups to
56  * return the specific stat value.
57  *
58  * To add a statistic, create the Stat variable in the component where it is
59  * being used and associate it with a StatGroup. The Stat infrastructure does
60  * the rest of the work for plumbing that statistic up to the public API
61  * class. Each API class must provide a getter method to access the specific
62  * statistic. Currently, this is done manually.
63  */

106     /**
107      * Add all the stats from the other group into this group. If both groups
108      * have the same stat, add the values.  The caller must make sure that no
109      * stats are added to or removed from the argument during this call.
110      *
111      * @throws ConcurrentModificationException if the addition or removal of
112      * stats in the argument is detected
113      */
114     @SuppressWarnings("unchecked")
115     public void addAll(StatGroup other) {
116 
117         for (Entry<StatDefinition, Stat<?>> entry :
118              other.stats.entrySet()) {
119 
120             StatDefinition definition = entry.getKey();
121             Stat<?> localStat;
122             synchronized (stats) {
123                 localStat = stats.get(definition);
124                 if (localStat == null) {
125                     stats.put(definition, entry.getValue());
126                     continue;
127                 }
128             }
129 
130             /*
131              * Cast to get around type problem. We know it's the same stat type
132              * because the definition is the same, but the compiler doesn't
133              * know that.
134              */
135             @SuppressWarnings("rawtypes")
136             Stat additionalValue = entry.getValue();
137             localStat.add(additionalValue);
138         }
139     }

149     public StatGroup computeInterval(StatGroup baseGroup) {
150         Map<StatDefinition, Stat<?>> intervalValues =
151                 new HashMap<StatDefinition, Stat<?>>();
152 
153         for (Entry<StatDefinition, Stat<?>> entry :
154              stats.entrySet()) {
155             StatDefinition definition = entry.getKey();
156             Stat<?> statValue = entry.getValue();
157             @SuppressWarnings("rawtypes")
158             Stat baseStat = baseGroup.stats.get(definition);
159             if (baseStat == null) {
160                 intervalValues.put(definition, statValue.copy());
161             } else {
162                 intervalValues.put(definition,
163                                    statValue.computeInterval(baseStat));
164             }
165         }
166         return new StatGroup(groupName, groupDescription, intervalValues);
167     }
