29 /**
30  * A StoppableThread is a daemon that obeys the following mandates:
31  * - it sets the daemon property for the thread
32  * - an uncaught exception handler is always registered
33  * - the thread registers with the JE exception listener mechanism.
34  * - its shutdown method can only be executed once. StoppableThreads are not
35  *   required to implement shutdown() methods, because in some cases their
36  *   shutdown processing must be coordinated by an owning, parent thread.
37  *
38  * StoppableThread is an alternative to the DaemonThread. It also assumes that
39  * the thread's run() method may be more complex than that of the work-queue,
40  * task oriented DaemonThread.
41  *
42  * A StoppableThread's run method should catch and handle all exceptions. By
43  * default, unhandled exceptions are considered programming errors, and
44  * invalidate the environment, but StoppableThreads may supply alternative
45  * uncaught exception handling.
46  *
47  * StoppableThreads usually are created with an EnvironmentImpl, but on
48  * occasion an environment may not be available (for components that can
49  * execute without an environment). In that case, the thread obviously does not
50  * invalidate the environment.
51  *
52  * Note that the StoppableThread.cleanup must be invoked upon, or soon after,
53  * thread exit.
54  */
55 public abstract class StoppableThread extends Thread {

144      * If the shutdown flag is false, set it to true and return false; in this
145      * case the caller should perform shutdown, including calling {@link
146      * #shutdownThread}. If the shutdown flag is true, wait for this thread to
147      * exit and return true; in this case the caller should not perform
148      * shutdown.
149      *
150      * When shutdownDone is initially called by thread X (including from the
151      * run method of the thread being shutdown), then a thread Y calling
152      * shutdownDone should simply return without performing shutdown (this is
153      * when shutdownDone returns true). In this case it is important that this
154      * method calls {@link #waitForExit} in thread Y to ensure that thread X
155      * really dies, or that an EnvironmentWedgedException is thrown if X does
156      * not die. In particular it is important that all JE threads have died and
157      * released their resources when Environment.close returns to the app
158      * thread, or that EWE is thrown if any JE threads have not died. This
159      * allows the app to reliably re-open the env, or exit the process if
160      * necessary. [#25648]
161      *
162      * Note than when thread X has sub-components and manages their threads,
163      * thread X's shutdown method will call shutdown for its managed threads.
164      * Waiting for exit of thread X will therefore wait for exit of its managed
165      * threads, assuming that all shutdown methods calls shutdownDone as
166      * described.
167      *
168      * @param logger the logger on which to log messages
169      *
170      * @return true if shutdown is already set.
171      */

279      * This method is invoked from another thread of control to shutdown this
280      * thread. The method tries shutting down the thread using a variety of
281      * techniques, starting with the gentler techniques in order to limit of
282      * stopping the thread on the overall process and proceeding to harsher
283      * techniques:
284      *
285      * 1) It first tries a "soft" shutdown by invoking
286      * <code>initiateSoftShutdown()</code>. This is the technique of choice.
287      * Each StoppableThread is expected to make provisions for a clean shutdown
288      * via this method. The techniques used to implement this method may vary
289      * based upon the specifics of the thread.
290      *
291      * 2) If that fails it interrupts the thread.
292      *
293      * 3) If the thread does not respond to the interrupt, it invalidates the
294      * environment.
295      *
296      * All Stoppable threads are expected to catch an interrupt, clean up and
297      * then exit. The cleanup may involve invalidation of the environment, if
298      * the thread is not in a position to handle the interrupt cleanly.
299      *
300      * If the method has to resort to step 3, it means that thread and other
301      * resources may not have been freed and it would be best to exit and
302      * restart the process itself to ensure they are freed. In this case an
303      * EnvironmentWedgedException is used to invalidate the env, and the EWE
304      * will be thrown when the app calls Environment.close.
305      *
306      * @param logger the logger on which to log messages
307      */
308     public void shutdownThread(Logger logger) {
309 
310         /*
311          * Save resource usage, since it will not be available once the
312          * thread has exited.
313          */
314         ThreadMXBean threadBean = ManagementFactory.getThreadMXBean();
315         if (threadBean.isThreadCpuTimeSupported()) {
316             totalCpuTime = threadBean.getThreadCpuTime(getId());
317             totalUserTime = threadBean.getThreadUserTime(getId());
318         } else if (threadBean.isCurrentThreadCpuTimeSupported() &&
319                    Thread.currentThread() == this) {
320             totalCpuTime = threadBean.getCurrentThreadCpuTime();
321             totalUserTime = threadBean.getCurrentThreadUserTime();
322         }
323 
324         if (Thread.currentThread() == this) {
325             /* Shutdown was called from this thread's run method. */
326             return;
327         }
328 
329         try {
330             LoggerUtils.info(logger, envImpl,
331                              getName() + " soft shutdown initiated.");
332 
333             final int waitMs = initiateSoftShutdown();
334 
335             /*
336              * Wait for a soft shutdown to take effect, the preferred method
337              * for thread shutdown.
338              */
339             if (waitMs >= 0) {
340                 join(waitMs);
341             }
342 
343             if (!isAlive()) {
344                 LoggerUtils.fine(logger, envImpl, this + " has exited.");
345                 return;
346             }
347 
348             LoggerUtils.warning(
349                 logger, envImpl,
350                 "Soft shutdown failed for thread:" + this +
351                     " after waiting for " +  waitMs +
352                     "ms resorting to interrupt.");
353 
354             interrupt();
355 
356             /*
357              * The thread must make provision to handle and exit on an
358              * interrupt.
359              */
360             final long joinWaitTime =
361                 (waitMs > 0) ? 2 * waitMs : DEFAULT_INTERRUPT_WAIT_MS;
362 
363             join(joinWaitTime);
364 
365             if (!isAlive()) {
366                 LoggerUtils.warning(logger, envImpl,
367                                     this + " shutdown via interrupt.");
368                 return;
369             }
370 
371             /*
372              * Failed to shutdown thread despite all attempts. It's
373              * possible that the thread has a bug and/or is unable to
374              * to get to an interruptible point.
375              */
376             final String msg = this +
377                 " shutdown via interrupt FAILED. " +
378                 "Thread still alive despite waiting for " +
379                 joinWaitTime + "ms.";
380 
381             LoggerUtils.severe(logger, envImpl, msg);
382             LoggerUtils.fullThreadDump(logger, envImpl, Level.SEVERE);
383 
384             if (envImpl != null) {
385                 @SuppressWarnings("unused")
386                 EnvironmentFailureException unused =
387                     new EnvironmentWedgedException(envImpl, msg);
388             }
389         } catch (InterruptedException e1) {
390             LoggerUtils.warning(
391                 logger, envImpl,
392                 "Interrupted while shutting down thread:" + this);
393         }
394     }

397      * Used to wait for thread shutdown, when {@link #shutdownDone} returns
398      * true because it has been called by another thread.
399      */
400     private void waitForExit(Logger logger) {
401 
402         assert shutdown.get();
403 
404         if (Thread.currentThread() == this) {
405             /* Shutdown was called from this thread's run method. */
406             return;
407         }
408 
409         try {
410             join(WAIT_FOR_SHUTDOWN_MS);
411 
412             if (!isAlive()) {
413                 return;
414             }
415 
416             /*
417              * For some reason, shutdown has not finished. This is unlikely,
418              * but possible. As in shutdownThread, we try interrupting the
419              * thread before giving up.
420              */
421             LoggerUtils.warning(
422                 logger, envImpl,
423                 "Soft shutdown failed for thread:" + this +
424                     " after waiting for " + WAIT_FOR_SHUTDOWN_MS +
425                     "ms, resorting to interrupt in wait-for-shutdown.");
426 
427             interrupt();
428             join(WAIT_FOR_SHUTDOWN_MS);
429 
430             if (!isAlive()) {
431                 return;
432             }
433 
434             /*
435              * Failed to shutdown thread despite all attempts. It's
436              * possible that the thread has a bug and/or is unable to
437              * to get to an interruptible point.
438              */
439             final String msg = this +
440                 " shutdown via interrupt FAILED during wait-for-shutdown. " +
441                 "Thread still alive despite waiting for " +
442                 WAIT_FOR_SHUTDOWN_MS + "ms.";
443 
444             LoggerUtils.severe(logger, envImpl, msg);
445             LoggerUtils.fullThreadDump(logger, envImpl, Level.SEVERE);
446 
447             if (envImpl != null) {
448                 @SuppressWarnings("unused")
449                 EnvironmentFailureException unused =

451             }
452         } catch (InterruptedException e1) {
453             LoggerUtils.warning(
454                 logger, envImpl,
455                 "Interrupted during wait-for-shutdown:" + this);
